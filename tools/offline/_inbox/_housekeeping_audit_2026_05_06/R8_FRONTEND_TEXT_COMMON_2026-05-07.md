**Defect found (FIXED in this commit)**:
- `sitemap-cities.xml` is a valid shard (21 valid URLs, header dated
  2026-05-04, generated by `scripts/generate_city_pages.py` — 20 政令指定都市
  + 1 index) but **was not referenced** by `sitemap-index.xml` nor declared
  in `robots.txt` Sitemap: directives. Googlebot / Bingbot would discover the
  21 city hub pages only via internal anchor crawl, never via the sitemap
  protocol. Fix: added single `<sitemap>` entry to `sitemap-index.xml` and
  one `Sitemap: https://jpcite.com/sitemap-cities.xml` line to `robots.txt`.
  Post-fix: `sitemap-index.xml` references 11 shards, robots.txt declares 12
  Sitemap: directives (11 shards + 1 master index). Symmetric difference
  between the two surfaces is now empty (modulo robots.txt also listing the
  master index by design).

**`.html` extension drift (UNRESOLVED — deferred to drift A)**:
- 441 `<loc>` entries still carry `.html`: sitemap-audiences (12) +
  sitemap-enforcement (300) + sitemap-prefectures (47) + sitemap-qa (82). CF
  Pages strips `.html` natively → 308 redirect. Functional but suboptimal for
  Googlebot canonical resolution. Top-level `sitemap.xml` (49 `.html` URLs)
  is by-design: pricing/about/etc. canonical IS `.html` (matches per-page
  `<link rel="canonical">`). Resolution path documented in
  `R8_SEO_DRIFT_DEEP_DIVE_2026-05-07.md` — requires generator-side rewrite
  (`generate_audience_pages.py`, `generate_enforcement_pages.py`,
  `generate_prefecture_pages.py`, `generate_qa_pages.py`), out of scope for
  trivial cross-page fix.
