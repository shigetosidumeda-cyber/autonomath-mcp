## 3. User journey — landing → pricing → key issue → first call

1. landing → hero CTA `Pricing で発行 →` で `/pricing.html#api-paid` へ。 redirect chain 308→200、 final URL `/pricing` (`.html` strip 慣習)。
2. pricing page H2 `id="api-paid"` に `¥3.30 / unit (税込)` 大字表示。 「通常検索・詳細取得は 1 unit。 batch/export は事前表示の式」「監査 workpaper 等固定 export fee は実行前に内訳表示」 で課金粒度の事前透明性 OK。
3. 鍵発行 CTA は pricing 内 `#api-paid` で完結 (signup.html は 404、 これが正しい設計 — Stripe checkout 経由)。
4. 既存鍵管理は `/dashboard.html` (308 → /dashboard, 200)。 新規 user 向け onboarding は `/docs/getting-started/` 200 / `/playground.html?flow=evidence3` 200 が hero から 1 click。
5. 法的 footer: `tos.html` `tokushoho.html` (特商法表記) `privacy.html` `legal-fence.html` 全 200。 `/legal-fence.html` は 6 業法 (税理士法 14 / 弁護士法 9 / 司法書士法 9 / 行政書士法 9 / 公認会計士法 9 / 社労士法 5) を 1 page 集約。

**friction list**:

- F1 (medium): `/terms` `/disclaimer` `/legal` を直接打つと 404。 hero 末尾と footer は `tos.html` `legal-fence.html` 文言のため、 user が一般語 (terms / 利用規約 / disclaimer) で URL 打つと迷子化。 → footer の表示文言は維持で可、 単に terms→tos / disclaimer→legal-fence の 308 redirect 1 行を CF Pages に追加すれば解消。
- F2 (small): hero-sub 200 字の jargon 密度。 「裏取り時間 短縮」「監査可能 引用」 の 2 句に短縮 → 詳細は below scroll に温存、 で 5-sec rule strengthen。
- F3 (small): 検索クエリ `/v1/programs?q=...` (without `/search`) が 404 だが 「指定パスは存在しません」+ `suggested_paths: [/v1/programs/search,...]` を返すため辿れる。 が `/v1/programs` で reach する user は多そう、 308 → `/v1/programs/search` redirect が gentler。
- F4 (none for now): 匿名 3 req/日が IP 共有環境 (社内 NAT・カフェ Wi-Fi) で誰かに使い切られる懸念。 429 response に「カードなしで試す (14 日 / 200 req)」 trial CTA が同梱されているため、 friction → upgrade path が natural に繋がっている。 改善提案 nil。

## 4. Error handling UX

| シナリオ | HTTP | Japanese clarity | retry / upgrade CTA |
|---|---|---|---|
| `/v1/nonexistent` | 404 | 「指定パスは存在しません。 https://api.jpcite.com/v1/openapi.json で有効なパス一覧を確認してください」 + suggested_paths 5 件 | documentation URL 明記 |
| `/v1/programs?q=...` | 404 | 同上、 suggested_paths に `/v1/programs/search` 含む | あり |
| anonymous 4-th req | 429 | 「匿名リクエスト上限 (3/日) に達しました。 明日また 3 回お試しいただけます (JST 翌日 00:00 リセット)。 X-API-Key ヘッダを設定すれば即解除されます」 | retry_after 27,196 sec / reset_at_jst / upgrade_url / direct_checkout_url / trial_signup_url / trial_terms (14 日 / 200 req / card 不要) ja+en 両方 |
| POST `/v1/programs/search` | 405 | 「このパスでは指定 HTTP メソッドは使えません。 Allow レスポンスヘッダで許可されているメソッドを確認してください」 | documentation URL |

全 error response に `request_id` / `severity` / `documentation` / `code` / ja+en `user_message` を envelope 同梱。 429 は trial onboarding まで 1 hop で繋げる設計、 業界比でも上位品質。

500 path: anonymous quota 枯渇のため再現できず (本来 quota 再 charge してから 500 を引き起こすには認証鍵が必要、 read-only 制約上スキップ)。 ENV 上 Sentry 接続済 (memory R7 audit より) のため運用側可視化は確保。

## 6. Verdict

- 🟢 first impression : 5-sec rule PASS、 hero に「Evidence Packet + ¥3.30/unit + 匿名 3 req/日」 が 1 画面で読み取れる。
- 🟢 cohort 7 page : 全 200 / JSON-LD 全 valid / 業法 fence 全表示 / placeholder 0 / mobile viewport OK。
- 🟢 pricing transparency : ¥3.30/unit が hero CTA + pricing H2 + 429 envelope の 3 surface に統一。 batch/export 内訳事前表示で「予期しない大額課金」リスク軽減。
- 🟢 error UX : 404/405/429 全て ja+en + retry/upgrade/trial の 3 ladder を 1 envelope に同梱。 業界トップ水準。
- 🟡 friction : F1 (terms/disclaimer 404) と F3 (`/v1/programs` 404) は user が "guess" で打つ確率が高い path、 308 redirect 追加で除去推奨。 ship blocker ではない。
- 🟢 業法 fence : 7 cohort + `/legal-fence` で 6 業法カバー、 「代行しない」「個別判断は資格者に」 文言が cohort 単位で明示。 詐欺リスク feedback (memory: feedback_no_fake_data) 適合。

ship 後 user 満足の主リスクは「hero-sub の jargon 密度」 1 点に集約。 それ以外は real persona walkthrough で blocker 検出されず。
