# MF アプリポータル 審査担当者向け 動作確認手順

## 前提

- 審査用 MF 事業者アカウント: `mf-review@bookyou.net` (パスワードは secure_notes 経由で別送)
- 審査用 jpcite.com 側 demo dataset: 公開済み制度データのみ。本番事業所と接続しません。
- プラグイン URL: https://mf-plugin.jpcite.com

## 1. OAuth 認可 (所要 30 秒)

1. プラグイン URL を MF クラウドの アプリポータル / アプリストア経由で起動する。
2. 「MF と連携する」ボタン (右上) をクリック → MF 認可画面に遷移。
3. 上記 demo アカウントでログイン → scope `mfc/ac/data.read` の同意画面が表示。
4. 同意 → プラグインに戻り、ヘッダに「事業者: (demo 事業者名)」と表示されることを確認。

## 2. 補助金検索タブ (所要 30 秒)

1. キーワード `省エネ 設備投資` を入力。
2. 都道府県は空欄でも可 (任意)。
3. 「検索」 → 5 件以内の候補が出典 URL 付きで表示される。
4. 各カードの「出典を開く」リンクが各省庁・自治体の一次情報源 (Aggregator では無い) へ遷移することを確認。

## 3. 税制優遇検索タブ (所要 30 秒)

1. キーワード `賃上げ促進` を入力 → 「検索」。
2. 「賃上げ促進税制」が結果に含まれ、適用期間と出典 (国税庁 等) が表示される。

## 4. インボイス登録番号確認タブ (所要 20 秒)

1. 登録番号欄に `T8010001213708` (Bookyou 株式会社の登録番号) を入力 → 「確認」。
2. 「Bookyou 株式会社」の名称と国税庁 公表情報への deep-link が表示される。
3. 不正な番号 (`T1234567890123` 等の存在しない番号) を入力 → 「該当する登録番号は見つかりませんでした」と表示。

## 5. 法令検索タブ (所要 20 秒)

1. キーワード `電子帳簿保存法 第7条` を入力 → 「検索」。
2. 該当条文の e-Gov 法令検索への deep-link が表示される (CC-BY 4.0 出典明記)。

## 6. 判例検索タブ (所要 20 秒)

1. キーワード `役員報酬 損金算入` を入力 → 「検索」。
2. 関連判例 5 件以内が裁判所判例検索への deep-link 付きで表示される。

## 7. 税理士法 §52 免責の常時表示 (所要 5 秒)

- 全タブで画面下部のフッターに `税理士法 §52 — 本サービスは情報提供のみを目的とし...` の免責が常時表示されていることを確認。
- 任意の検索結果の生 JSON を確認すれば `_disclaimer` フィールドが付与されていることが確認できる。

## 8. ログアウト (所要 10 秒)

- ヘッダのログアウト操作 (またはセッション期限切れ) で MF 側 revoke endpoint が呼ばれ、再度 `/oauth/authorize` に戻ることを確認。

## 取得情報の確認

ブラウザ DevTools → Application → Cookies で `jpcite_mf_sid` (HttpOnly) があるのみ。
localStorage / sessionStorage / window.* に access_token / refresh_token が**漏れていない**
ことが確認できます。

## 想定される質問

- **Q. 仕訳データを読みますか?** → 読みません。`mfc/ac/data.read` を要求しますが、本アプリで実際に呼ぶのは事業者表示名と tenant_uid のみです。
- **Q. MF 利用者に直接課金しますか?** → しません。Bookyou が保有する jpcite.com のサービス契約に対して 1 req ¥3.30 (税込) の metered subscription として課金されます (3 req/日 無料)。
- **Q. データ residency は?** → 日本国内 (Fly.io HND リージョン + Cloudflare Tokyo edge)。
