# 業務効率化と業法 fence の両立 ——「jpcite」が示す 7 業法 disclaimer 設計

会計事務所 DX において、業務効率化は業法 fence と常に綱引きの関係にある。月額 ¥5,940 で 100 社月次 review を回す公的情報 AI「jpcite」は、税理士法 §52 を含む 7 業法に対する disclaimer 階層を機械的に組み込み、効率と適法性の同時成立を実証している。

## 7 業法と会計事務所 AI の壁

会計事務所が AI を業務に組み込むとき、最初に立ちはだかるのは「業法 fence」である。AI ベンダーが提供する出力が、独占業務を侵食するかどうか ——この境界判断は経営者にとって最大の不確実性となる。

jpcite が射程に入れる業法は次の 7 本である。

1. 税理士法 §52 (税務代理・税務書類作成・税務相談独占)
2. 弁護士法 §72 (法律事務独占)
3. 行政書士法 §19 (官公署提出書類作成独占)
4. 司法書士法 §73 (登記・供託代理独占)
5. 社会保険労務士法 §27 (労務関係書類作成独占)
6. 中小企業診断士・中小企業支援法 (公的伴走業務の建付け)
7. 金融商品取引法 §29 (投資助言業登録)

公的情報を機械整形して提供する artifact 型サービスは、これらの fence を犯さない設計が組み込まれていない限り、事務所側で導入リスクを引き受ける形となる。

## artifact という設計原理

jpcite は Bookyou 株式会社 (T8010001213708) が運営する公的情報 API・MCP サーバである。中小企業庁・経済産業省・国税庁・e-Gov・各都道府県・独立行政法人の公開情報を、機械可読な artifact (構造化 JSON + 出典 URL + 整形 Markdown) に変換して提供する。

ここで重要なのは、jpcite が「個別顧客の事実関係を前提とした判断」を一切出力しないことである。

出力されるのは常に次の三点セットである。

- 公開情報の機械整形 (構造化テキスト)
- 出典 URL (一次情報への back-link)
- 該当しうる制度の候補リスト (絞り込みは事務所側)

この設計により、jpcite の出力は「公開情報の整形」に留まり、税理士業務・行政書士業務・社労士業務などの独占業務を構成しない。

## 7 業法 disclaimer 階層

jpcite は、API request の context に応じて 7 業法それぞれに対応した disclaimer を機械的に付与する。

具体的には、レスポンスヘッダ `X-Jpcite-Disclaimer` と JSON body の `_disclaimer` field に、該当業法の参照と「最終判断は資格専門家へ」の文言を載せる。recipe r03 (M&A DD) であれば弁護士法・税理士法・金商法の 3 本、recipe r07 (認定支援機関業務) であれば中小企業支援法・税理士法の 2 本、というように、recipe 単位で fence 対象業法が明示されている。

さらに、jpcite SDK・MCP server は、disclaimer を必ず end-user に表示する義務を Terms of Service に盛り込み、再配布事業者にも継承させる。これにより、最終利用者 (顧問先) まで disclaimer chain が届く設計となっている。

## TKC ユーザにとっての位置づけ

TKC の OMS・FX シリーズは、会計事務所の「文書管理・申告書作成・税効果計算」をカバーする統合プラットフォームである。jpcite はこれと競合しない。むしろ補完する。

TKC が提供する内部処理 (顧問先データの構造化、申告書生成) に対し、jpcite は「外部公開情報の構造化された取得」を担う。OMS のワークフローに jpcite API を組み込めば、「顧問先別補助金 candidate を月次 alert として配信」「税制改正適用 eligibility を OMS の顧問先 master と join」といったワークフローが自動化できる。

TKC API・MCP 連携については、jpcite 側の OpenAPI v0.3.4 (182 endpoint) を参照し、OMS の外部連携設計に組み込む形が現実的である。recipe r01 / r07 は TKC ユーザの月次運用フローと相性が良い。
