# 中小 M&A の公開情報 DD を 5 分で artifact 化する手順 ——「jpcite」recipe r03 の実装

中小 M&A の Due Diligence (DD) において、買い手・売り手双方の公開情報を機械整形して artifact 化する API・MCP サーバ「jpcite」が、DD リードタイムを数日から 5 分に圧縮している。recipe r03 は弁護士法 §72・金商法 §29・税理士法 §52 の 3 業法 fence を組み込み、外部アドバイザーの最終判断業務を侵食しない設計を貫いている。

## 中小 M&A DD の現状課題

中小 M&A 市場は、事業承継需要の拡大に伴い拡大基調にある一方、DD 工程の効率化は遅々として進んでいない。理由は明確で、DD で扱う情報の多くが「公開情報の人手整形」に依存しているからである。

具体的には、対象企業について次の情報を毎案件で人手収集している。

- 登記情報・代表者経歴 (法務局・国税庁法人番号 portal)
- 業種別の許認可保有状況 (各省庁許認可 DB)
- 関連業法の改正履歴 (e-Gov)
- 補助金採択履歴 (中小企業庁・各都道府県)
- 入札参加資格・指名停止履歴 (各省庁・自治体)
- 業界相場・類似案件のマルチプル (公開 M&A DB)

中堅 M&A アドバイザリーで 1 案件あたり 20〜40 時間が公開情報の人手整形に消費されており、有資格者 (弁護士・税理士・FAS) の稼働時間を圧迫している。これを artifact レイヤで肩代わりするのが jpcite recipe r03 である。

## recipe r03 の入出力

jpcite は Bookyou 株式会社 (T8010001213708) が運営する公的情報 API・MCP サーバである。recipe r03「M&A 公開情報 DD」は、対象企業の法人番号・業種コード・所在地を input として、構造化 artifact を 5 分以内に出力する。

artifact の構成:

- 登記情報の機械整形 (法人番号 portal 出典)
- 許認可保有制度の候補リスト (省庁 DB 横断)
- 業界該当業法の改正履歴 (直近 36 ヶ月)
- 補助金採択履歴 (採択日・制度名・金額帯)
- 入札参加履歴 (公表案件のみ)
- 関連 e-Gov 法令本文への back-link

すべての情報に出典 URL が付与され、買い手側がそのまま一次情報を辿れる。recipe r03 の実装手順と payload 仕様は https://jpcite.com/docs/recipes/r03 で公開している。

## 3 業法 fence の設計

中小 M&A DD は、弁護士業務・税理士業務・金融商品取引業を同時に視野に入れた fence 設計が必要となる。jpcite は次の三層で防護している。

第一に、recipe r03 の出力は「公開情報の機械整形」に限定し、買収条件の助言・スキーム選択の推奨・mulipal の判断・契約書ドラフトを一切生成しない。これにより弁護士法 §72 (法律事務独占)、税理士法 §52 (税務代理・税務相談独占)、金商法 §29 (投資助言業登録) を侵食しない。

第二に、API response に `X-Jpcite-Disclaimer` header と `_disclaimer` field が付与され、「公開情報の機械整形であり、M&A スキーム判断・税務助言・投資助言は資格専門家・登録業者の業務であること」を明示する。

第三に、SDK / MCP server は disclaimer chain を再配布事業者にも継承させ、最終利用者 (買い手・売り手) まで「artifact は事実整理にとどまる」事実が届く設計とする。

M&A アドバイザリーが jpcite を導入する場合、「artifact を DD 事前整理に用い、スキーム判断・条件交渉・契約書作成は弁護士・税理士が担う」という業務分担を保つ限り、3 業法 fence を犯さない。

## 節約額試算 (旧称 ROI 試算)

中堅 M&A アドバイザリーで月 5 案件を扱うケース。

- 従来: 案件あたり DD 事前整理 25 時間 × 5 件 × 有資格者単価 ¥10,000 = 月額 ¥1,250,000
- jpcite 導入後: API ¥600 + 確認・統合 5 時間/件 × 5 件 × ¥10,000 = 月額 ¥250,600

月額 ¥999,400 のキャッシュ改善、年間 ¥12 百万円。さらに DD リードタイムが圧縮されることで、案件回転率が向上し、年間取扱件数の増加が期待できる。

## 5 分で artifact 化する実装手順

recipe r03 の実装は以下の最小構成で可能である。

1. jpcite アカウント発行 (info@bookyou.net)
2. API key 取得、SDK 導入 (npm / pip / go)
3. 対象法人の法人番号を input、recipe r03 を call
4. JSON artifact を Markdown / PDF / Slack 投稿に変換
5. 有資格者が artifact を review し、DD 報告書本体に統合

MCP server 経由であれば、Claude Desktop / Cursor IDE から自然言語で「法人番号 XXX の M&A DD artifact を出して」と指示するだけで artifact が呼び出せる。価格は ¥1/request、月額契約で daily 3 request 無料、超過分は完全従量。recipe r03 は約 ¥120/案件 (request 数依存) で完結する。
