# 許認可 eligibility chain の自動化 ——「jpcite」が示す行政書士法 §19 を犯さない設計

許認可案件の eligibility 評価で使う公的情報を artifact 化する API「jpcite」が、行政書士業務の月次 review に必要な調査材料をまとめる。recipe r05 は許認可 chain 上の公開情報を機械整形して提供する一方、行政書士法 §19 の fence を犯さない artifact 設計を貫いている。

## 許認可 eligibility chain の構造

行政書士業務における許認可案件は、単独制度の単発申請ではなく、複数の制度を chain 状に串刺しした構造を持つ。

例として、飲食店開業のケースを取る。

- 食品衛生法に基づく飲食店営業許可
- 風営法に基づく深夜酒類提供届出 (営業時間次第)
- 防火対象物使用開始届
- 屋外広告物条例の許可
- 食品事故補償保険の加入義務
- 各自治体の独自上乗せ条例

依頼者一人あたり、平均 6〜10 制度の eligibility を chain 状に評価することになる。この評価には、各制度の公開情報 (法令本文・施行規則・自治体上乗せ・最新通達) を横断する作業が含まれ、相談初期段階で 1 案件あたり 2〜4 時間が消費される。

行政書士業務の本丸は「申請書類の作成と提出代理」だが、eligibility chain の事前整理が業務時間の 4 割を占めている事務所も少なくない。ここに AI を投入する余地が大きい。

## artifact として提供される公開情報

jpcite は Bookyou 株式会社 (T8010001213708) が運営する公的情報 API・MCP サーバである。e-Gov 法令データベース、各省庁の通達・FAQ、自治体の許認可 portal、独立行政法人の解説資料を、構造化 JSON + 出典 URL の artifact に変換して提供する。

recipe r05「許認可 eligibility chain 評価」は、業種・所在地・営業形態を input として、関連する許認可制度の候補リスト + 各制度の根拠法令 + 申請窓口 + 標準処理期間 + 出典 URL を構造化 artifact として出力する。

artifact 仕様:

- 関連法令の本文抜粋 (e-Gov 出典)
- 自治体上乗せ条例の検出 (該当があれば候補として列挙)
- 申請窓口 URL
- 標準処理期間 (各窓口公表値)
- 添付書類リスト (申請手引きから抽出)
- 改正履歴 (直近 12 ヶ月)

## 行政書士法 §19 fence の設計

ここで最も重要なのは、jpcite が「官公署に提出する書類の作成」を一切行わないことである。

行政書士法 §19 は、官公署提出書類の作成業務を行政書士の独占とする。AI ベンダーが申請書類のドラフトを生成すれば即座に同条違反となる。jpcite はこの fence を次の三層で防護している。

第一に、recipe r05 の出力は「許認可制度の候補リスト + 根拠 + 申請窓口情報」までであり、申請書類のドラフトを一切生成しない。記入例・記入欄候補等の出力も recipe 仕様で禁止されている。

第二に、申請・許認可・専門判断に関わる sensitive surface では、`X-Jpcite-Disclaimer` header や JSON body の `_disclaimer` field により、「公開情報の機械整形であり、書類作成・申請代理は行政書士業務であること」を示す。

第三に、jpcite SDK と MCP server は、再配布事業者にも disclaimer chain の継承を ToS で義務付ける。最終利用者まで「artifact は事実整理にとどまる」という事実が届く仕組みである。

行政書士事務所が jpcite を導入する際は、「artifact を eligibility chain の事前整理に用い、書類作成と提出代理は行政書士が担う」という業務分担を保つ限り、§19 fence を犯さない。

## 節約額試算 (旧称 ROI 試算)

許認可案件 月 20 件を扱う中堅事務所のケース。

- 従来: 案件あたり eligibility 評価 3 時間 × 20 件 × 時給 ¥5,000 = 月次 API fee delta
- jpcite 導入後: API ¥1,200 + 確認 30 分/件 × 20 件 × ¥5,000 = 月次 API fee delta

月次 API fee delta のAPI fee delta reference、年間参考額。さらに事前整理時間が圧縮されることで、相談初回から見積提示までのリードタイムが半減し、業務成果は本比較に含みません。

レシピ r05「許認可 eligibility chain 評価」の実装手順と recipe payload 仕様は https://jpcite.com/docs/recipes/r05 で公開している。Claude Desktop / Cursor IDE から MCP server 経由で自然言語で artifact を引き出すことも可能。

## 行政書士業務の中核は変わらない

jpcite が自動化するのは、許認可案件の前処理・公開情報整理の部分である。書類作成・申請代理・依頼者ヒアリング・自治体折衝といった、行政書士業務の本質的な価値創出は変わらず行政書士が担う。

むしろ前処理を圧縮することで、依頼者一人あたりに割ける相談時間が増え、案件単価の上昇や付加サービスの拡張に繋がる。AI を「業務を奪う存在」ではなく「事前整理を肩代わりする存在」と位置付ければ、行政書士業務の業務成果は本比較に含みません。
