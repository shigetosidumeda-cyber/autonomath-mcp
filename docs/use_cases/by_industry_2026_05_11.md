# 業種別 jpcite Use Case (2026-05-11)

¥3/billable unit (税込 ¥3.30)、3 req/日 anonymous free per IP、tier/seat/年間最低額なし。
本ドキュメントは jpcite が **どの業種の AI agent から、どんな業務 flow で呼ばれ、どれだけの ROI を生むか** を実務シナリオで具体化する。
すべての利用は AI agent 経由 (Claude Desktop / ChatGPT GPT / Cursor / Codex / 顧問先別 Slack bot etc.)。jpcite.com の人間 UI で士業本人がブラウズする使い方は対象外。

corpus snapshot (CLAUDE.md SOT 2026-05-07):
programs 11,601 (S 114 / A 1,340 / B 4,186 / C 5,961) + 採択 2,286 + 融資 108 (担保/個人保証人/第三者保証人 三軸) + 行政処分 1,185 + 法令 6,493 全文 + 9,484 catalog + 50 tax_rulesets + 通達 3,221 + 国税不服審判所 裁決 137 + 適格事業者 13,801 delta (月次 4M 行 bulk wired) + 排他/前提ルール 181 + am_compat_matrix 43,966。MCP 139 tool 公開 (146 runtime)。

## 1. 税理士事務所 (顧問先月次レビュー / 補助金提案)

**painpoint**: 顧問契約は月額 ¥3 万-¥10 万/社 (TKC 報酬規程・税理士法人実勢) で、顧問先 50-150 社を 1 税理士が担当。最大の解約理由は「使える特例を見落とされた」「補助金の存在自体を知らなかった」の不作為クレーム。特に賃上げ促進税制 (措置法 §42-12-5)・中小企業投資促進税制 (§42-6)・DX 投資促進税制・電帳法改正・インボイス 2 割特例の周辺は年に 4-6 回改正が走り、紙の改正速報を全顧問先に当てるのは非現実的。

**現状の代替**: TKC 戦略経営者通信 / 税務通信 / 国税庁ホームページ巡回。改正タイミングと顧問先ごとの該当判定は税理士が頭で当てる。

**問題**: 顧問先 100 社全てに該当判定を当てると 1 社 5 分でも 8 時間/月、人手で読み落としが必ず出る。発覚は税務調査時 = 信頼回復不能。

**jpcite 利用 sequence (Claude Desktop / 顧問先別 Slack bot)**:
- 月次レビュー: 顧問先ごとに `prepare_kessan_briefing` (¥3) で 前月 決算窓口 から am_amendment_diff + jpi_tax_rulesets を抽出。100 社 × ¥3 = **¥300/月**。
- 提案時: `search_tax_incentives` + `apply_eligibility_chain_am` で措置法 §42 系の該当判定 + 排他ルール check (1 顧問先あたり 3-5 req)。月 10 提案で +¥150。
- 改正発生時: 保存検索 (`saved_searches.profile_ids`, mig 097) + `dispatch_webhooks.py` で Slack alert (顧問契約に乗っているので別請求不可、月内に自動配信)。

**月額**: 顧問先 100 社 × ¥3 + 提案 ¥150 = **¥450/月 (税別)**。
**回避できる失敗コスト**: 取りこぼし 1 件 = 顧問契約解除 1 件 = ¥3-¥10万/月 × 12 = **¥36-¥120 万/年 ロス**。
**ROI 倍率**: ¥450 × 12 = ¥5,400/年 投資で ¥36 万-¥120 万/年 リスク回避 = **67 倍-222 倍**。
**agent platform**: Claude Desktop (顧問先別 X-Client-Tag 分離) + audit_seal pack (mig 089) で月次自動 PDF。

## 2. 会計士事務所 (監査対応 / 研究開発税制 / IT導入会計処理)

**painpoint**: 監査契約 1 社あたり ¥500 万-¥3,000 万/年 (日本公認会計士協会 監査報酬指針)。監査ファイル準備で「会計処理選択肢の根拠条文」「適用ガイドライン」を被監査会社の経理に確認する往復が毎期 50-200 件。特に 研究開発税制 (措置法 §42-4) は 2024 年税制改正で控除上限/オープンイノベーション要件が変わり、IT 導入補助金の会計処理 (補助金特別勘定 / 直接控除) は実務指針 No.30 と税効果会計の交差で監査側のチェック工数が膨れる。

**現状の代替**: 監査六法・会計監査ジャーナル・JICPA 会計制度委員会報告。法人別に該当判定するには監査チームが手探り。

**問題**: 紙ベース照会で 1 論点 30-60 分 × 50 件/期 = 25-50 時間。監査報酬時間単価 ¥15,000 換算で ¥37 万-¥75 万のロス。被監査会社からの「監査対応コストが高い」苦情 → 監査契約見直し圧力。

**jpcite 利用 sequence (Cursor 監査用 prompt / Claude Desktop)**:
- 論点照会: `get_am_tax_rule` (¥3) で措置法 §42-4 ruleset + 通達 link + 経理処理ガイドライン 1 ストロークで取得。
- 被監査会社全体 sweep: `bundle_application_kit` + `match_due_diligence_questions` (Wave 22, ¥3 × 2 = ¥6) で IT 導入補助金 受給先一覧から論点 deck 30-60 問を自動 assemble。
- 監査ファイル 1 社あたり 20-40 req = **¥60-¥120/社/期**。10 社担当チームで **¥600-¥1,200/期**。

**月額**: 監査 10 社 × ¥120/期 ÷ 3 ヶ月 = **¥400/月 (税別)** ピーク期、平時 ¥100/月。
**回避できる失敗コスト**: 論点照会工数 25 時間/期 × ¥15,000 = **¥37.5 万/期 工数削減** + 被監査会社満足度維持で監査契約 ¥500 万-¥3,000 万/年 の更新確度 +5pt。
**ROI 倍率**: ¥1,200/期 投資で ¥37.5 万/期 削減 = **312 倍** (時間価値ベース、契約更新確度効果は別)。
**agent platform**: Cursor (監査調書テンプレ生成) + Claude Desktop (論点照会)。
※ 弁護士法 §72 / 公認会計士法 §47条の2 で監査意見そのものは jpcite 不可、scaffold + 一次 URL のみ。

## 3. 行政書士 (建設業許可 / 産廃許可 / 在留資格 / 補助金申請)

**painpoint**: 申請 1 件あたり報酬 ¥10 万-¥30 万 (日行連 標準額)、建設業 新規許可で ¥15 万前後。事前リサーチで「許認可 + 関連補助金 + 行政処分歴 + 排他ルール」を当てる工数が 1 件 3-6 時間。許可取得後に「同時に取れた補助金を知らなかった」「業者の過去処分を見落として代理した」事故が起きると懲戒対象 (行政書士法 §13 / §14)。

**現状の代替**: 建設業ハンドブック・産廃六法・国交省 / 環境省 portal・jGrants 巡回・帝国データバンク有償照会。

**問題**: 横断検索ができず、特に多業種の法人 (建設 + 産廃 + 運送) では 3-4 portal を別々に walk。事務所サイズによっては事務員 1 人で 月 20-30 件処理しきれず案件 cap 発生。

**jpcite 利用 sequence (Claude Desktop)**:
- 事前リサーチ: `intel_houjin_full` + `check_enforcement_am` (¥3 × 2 = ¥6) で 法人番号 → 過去 5 年処分歴 + 業務範囲。
- 関連補助金 sweep: `pack_construction` (Wave 23, JSIC D, ¥3) で建設業向け制度 top 10 + 国税不服審判所 裁決 + 通達 1 リクエスト。
- 排他確認: `check_exclusions` (¥3) で併給不可候補。
- 1 案件 10-20 req = **¥30-¥60/案件**。

**月額**: 月 25 案件 × ¥45 = **¥1,125/月 (税別)**。
**回避できる失敗コスト**: 行政処分歴見落としで懲戒 1 件 = 業務停止 30 日 = **月収 ¥80-¥150 万 ロス + 廃業リスク**。補助金取りこぼし 1 件 = 顧客からの後発クレーム + 紹介経路喪失 ¥30-¥50 万 LTV。
**ROI 倍率**: ¥1,125 × 12 = ¥13,500/年 投資で 懲戒 1 回回避 ¥80 万 = **59 倍**、紹介喪失 ¥50 万 LTV 回避 = **37 倍**。
**agent platform**: Claude Desktop (許認可 1 案件 ワンショット) + Cursor (定型 提案書テンプレ生成、jpcite は scaffold のみ、申請書面 generation は行政書士法 §1 で fence)。

## 4. 中小企業診断士 (経営診断 / 補助金診断 / 事業計画)

**painpoint**: 認定経営革新等支援機関ロールで補助金 申請支援 1 件 ¥30 万-¥100 万 (中小企業庁 認定機関 業界相場)、事業計画策定 ¥20 万-¥80 万。最大の競争軸は「使える制度の発見数 × 採択率を出せる根拠の量」。診断士 1 人で顧問先 30-80 社、各社の業種 × 都道府県 × 設備投資計画 × 経営状況から該当制度を当てる工数が膨大。採択率を盛って提案して落選すると顧客信頼失墜 → 報酬返還圧力。

**現状の代替**: 中小機構 J-Net21・ミラサポ plus・jGrants 検索。47 都道府県 ✕ 1,741 市区町村 ✕ 11,601 制度から手で探すのは非現実的。

**問題**: 制度の存在自体を見落とし → 顧問先から「他の診断士に頼めば良かった」と転出。制度発見数が ARR ロス直撃。

**jpcite 利用 sequence (Claude Desktop + 顧問先別 Slack bot)**:
- 月次 sweep: 顧問先 50 社 × `saved_searches.profile_ids` (mig 097) で run_saved_searches.py が cron 配信。fan-out 50 req = **¥150/月**。
- 個社診断: `find_complementary_programs_am` + `apply_eligibility_chain_am` + `forecast_program_renewal` (Wave 22) で 補完候補 + 排他 chain + 来期更新確度 を 3 req × ¥3 = ¥9/社。月 10 社診断で +¥90。
- 提案時: `bundle_application_kit` で kit assembly (¥3/件、scaffold のみ、申請書 generation は行政書士法 §1 で fence)。10 件 = ¥30。

**月額**: ¥150 + ¥90 + ¥30 = **¥270/月 (税別)**。
**回避できる失敗コスト**: 補助金提案 1 件取りこぼし = ¥30-¥100 万報酬ロス + 顧問契約 (月額 ¥5-¥15 万) 解約 1 件 = **年間 ¥60-¥180 万ロス**。
**ROI 倍率**: ¥270 × 12 = ¥3,240/年 投資で ¥60-¥180 万回避 = **185 倍-555 倍** (最高 ROI)。採択率公表値だけ表示する jpcite 設計 (推測値出さない) は診断士の信頼維持と整合。
**agent platform**: Claude Desktop (個社診断) + 顧問先別 Slack bot (月次 alert)。

## 5. M&A advisor (DD / 法令 compliance / 業法 fence)

**painpoint**: M&A 仲介 1 件 報酬レーマン方式 譲渡対価 5 億円で約 ¥2,500 万 (中小企業庁 M&A 支援機関登録制度 標準)。DD 期間 4-8 週間に対象会社 1 社あたり 法令・業法・行政処分・補助金返還義務・適格事業者登録・jurisdiction 不一致 を網羅する工数が深い。特に許認可業種 (建設・産廃・運送・介護・医療) の取得・承継要件、補助金受給歴に紐づく目的外使用禁止期間の残存、jurisdiction 不一致 (登記簿本店 vs 営業実態) は買収後に発覚すると条件付き合意の再交渉 = ディール崩壊リスク。

**現状の代替**: TDB / TSR 法人照会、紙ベース 業法チェックリスト、JFC・経産省 portal 巡回、弁護士事務所への都度照会 (1 件 ¥30 万-¥100 万)。

**問題**: 法人 1 社の DD で 50-200 軸を別々に集める。aggregator 詐欺 (補助金ポータル系の古い情報) も交じり信頼性のばらつき大。

**jpcite 利用 sequence (Cursor DD prompt / Claude Desktop)**:
- houjin 360°: `intel_houjin_full` + `cross_check_jurisdiction` + `check_enforcement_am` (Wave 22, ¥3 × 3 = **¥9**) で本店 vs 営業所 vs 適格事業者 jurisdiction + 行政処分 fine/grant_refund/subsidy_exclude。
- DD deck: `match_due_diligence_questions` (Wave 22, ¥3) で industry × portfolio × 与信 risk から 30-60 問の DD 質問 deck 自動 assemble。
- 周辺制度: 業種別 `pack_construction` / `pack_manufacturing` / `pack_real_estate` (Wave 23, 各 ¥3)。
- 1 ディール = mcp.json cost_examples 通り 約 47 req = **¥155/ディール (税込)**。

**月額**: 月 3-5 ディール × ¥155 = **¥465-¥775/月 (税込)**。
**回避できる失敗コスト**: jurisdiction 不一致見落としで条件再交渉 → ディール崩壊 = 報酬 **¥2,500 万** 喪失 1 件。補助金返還義務見落としで PMI 後の対象会社の隠れ債務 = ¥500 万-¥5,000 万。
**ROI 倍率**: ¥775 × 12 = ¥9,300/年 投資で ディール崩壊 1 件回避 ¥2,500 万 = **2,688 倍** (本ドキュメント中 最高)。
**agent platform**: Cursor (DD レポート自動 assemble) + Claude Desktop (個別法人照会)。
※ 弁護士法 §72 で法律判断は jpcite 不可、scaffold + 一次 URL のみ、最終判断は弁護士手元。

## 6. 信用金庫渉外担当 (顧客 watch / 制度紹介 / 融資前 due diligence)

**painpoint**: 254 信用金庫 × 担当 1 人あたり取引先 80-200 社。融資審査 1 件平均額 ¥1,000 万-¥1 億、貸出残高に対する不良債権率 1% 圧縮で支店利益 ¥数百万改善。融資前 due diligence で「補助金交付対象になりそうな設備投資なら金利優遇 + 信用保証協会推薦書面」、「過去行政処分歴」、「マル経推薦の併給可否」、「47 信用保証協会 連動」を当てる工数が高い。経営支援部門 (認定経営革新等支援機関ロール) は別途、経営革新計画認定の周辺制度を顧客に案内する義務。

**現状の代替**: SCBRI 公開資料 + 中小機構 / 経産省 portal 個別 walk + 業界誌 (信金協会報・銀行実務)。本部の制度通達は紙、支店渉外の頭に乗らない。

**問題**: 取引先 100 社全てに月次 sweep は不可、結果として制度紹介の網羅率 30-40%、競合地銀に取引を奪われる ARR ロス。

**jpcite 利用 sequence (Claude Desktop)**:
- 取引先別 watch: `houjin_watch` (mig 088) + `dispatch_webhooks.py` で 取引先 100 社の corp amendment + 補助金採択 alert を Slack 即時配信 (1 通 ¥3 で従量、月 30 alert で +¥90)。
- 融資前 DD: 1 社 1 call `loan-programs/search?collateral_required=not_required` (¥3) + `check_enforcement_am` (¥3) + `pack_*` (¥3) = 3-5 req。月 50 社 sweep で **¥450-¥750**。
- 顧問先 attribution: X-Client-Tag (mig 085) で支店内で顧客別費用按分。

**月額**: alert ¥90 + sweep ¥600 = **¥690/月 (税別)** = ¥759/月 (税込)。担当 1 人あたり 月 100-200 取引先 lookup なら ¥330-¥660/月 (mcp.json `1 担当者 月 100 lookup = ¥330/月 (税込)` と整合)。
**回避できる失敗コスト**: 競合地銀への取引離脱 1 件 = 平均融資残高 ¥5,000 万 × 利鞘 1% = **¥50 万/年 ロス**。不良債権発生 1 件 (DD 漏れによる) = 引当金 ¥数百万-¥数千万。
**ROI 倍率**: ¥759 × 12 = ¥9,108/年 投資で取引離脱 1 件回避 ¥50 万 = **55 倍**、引当金 ¥500 万回避 = **549 倍**。
**agent platform**: Claude Desktop (営業推進部 / 経営支援部 個別 lookup) + Slack webhook (本部から 支店渉外へ alert 配信)。

## ROI 倍率まとめ (最低-最高)

| 業種 | 月額 (税別) | 主要 回避コスト | ROI 倍率 |
|---|---|---|---|
| 税理士事務所 | ¥450 | 顧問契約解除 ¥36-¥120 万/年 | 67-222 倍 |
| 会計士事務所 (ピーク期) | ¥400 | 監査工数 ¥37.5 万/期 | **312 倍** |
| 行政書士 | ¥1,125 | 懲戒 ¥80 万 / 紹介喪失 ¥50 万 | 37-59 倍 (最低 ROI) |
| 中小企業診断士 | ¥270 | 顧問契約解除 ¥60-¥180 万/年 | 185-555 倍 |
| M&A advisor | ¥465-¥775 | ディール崩壊 ¥2,500 万 | **2,688 倍 (最高 ROI)** |
| 信用金庫渉外 | ¥690 | 引当金 ¥500 万 | 55-549 倍 |

説得力ある case (構造の堅さ × 数字の妥当性 × 業法 fence の整合): **M&A advisor (2,688 倍、cost_examples DD ¥155 と整合) + 中小企業診断士 (555 倍、saved_searches fan-out が cohort revenue model #5 と完全整合) + 信用金庫 (mcp.json 経営支援部担当 月 100 lookup = ¥330/月 と整合)** の 3 業種。

最低 ROI は行政書士 (37 倍) だが回避コストの確実性が高く、月額も最大 (¥1,125)。最低 ROI でも一般 SaaS の費用対効果を大幅超過。

すべての case で:
- LLM API 呼び出しは jpcite 側で **しない** (顧客側 Claude / Cursor が推論)
- 業法 fence は scaffold + 一次 URL のみ、最終判断は有資格者 (税理士法 §52 / 弁護士法 §72 / 行政書士法 §1 / 司法書士法 §3 / 公認会計士法 §47条の2)
- DPA / 営業 / CS 一切なし、API キー 1 本で完結 (Solo + zero-touch ops)
