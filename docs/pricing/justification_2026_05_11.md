# jpcite 料金の考え方

jpcite の標準単価は **1 billable unit = ¥3 税別 / ¥3.30 税込** です。通常の API / MCP 呼び出しは 1 unit として扱います。匿名枠は **3 req/日/IP** で、登録前にレスポンス形式と出典の出方を確認できます。

このページは、料金を判断するときに見るべき前提を公開向けに整理したものです。売上影響や機会損失を保証する説明ではありません。

## 料金の基本

| 項目 | 内容 |
|---|---|
| 通常単価 | ¥3 / billable unit 税別 |
| 税込単価 | ¥3.30 / billable unit |
| 無料枠 | 匿名 3 req/日/IP |
| 課金方式 | 従量課金 |
| 最低利用額 | なし |
| 契約期間 | なし |
| 主な用途 | AI agent からの一次出典付き検索、照合、scaffold 作成 |

例:

| 利用量 | 税別 | 税込 |
|---:|---:|---:|
| 100 units | ¥300 | ¥330 |
| 1,000 units | ¥3,000 | ¥3,300 |
| 10,000 units | ¥30,000 | ¥33,000 |

## 何に料金がかかるか

jpcite は、AI agent が日本の公的制度・法令・法人情報・行政処分などを扱う前段で、次のような情報を返します。

- 出典 URL
- 取得時点
- known gaps
- 互換・排他ルール
- response envelope
- 専門判断を避けるための注意文

LLM 推論そのものは jpcite 側では行いません。Claude、ChatGPT、Cursor、Codex など利用者側の AI agent が jpcite の REST / MCP API を呼び出します。

## 無料枠で確認すること

登録前は、匿名 3 req/日/IP の範囲で次を確認してください。

1. 出典 URL が省庁・自治体・公的機関などの一次資料に寄っているか
2. 取得時点や known gaps がレスポンスに出るか
3. 自分の AI agent から呼び出しやすい形式か
4. 1 回の業務で何 req くらい必要か
5. 有料利用時に `X-Cost-Cap-JPY` や月次上限で費用を止められるか

## よくある利用量の目安

| 用途 | 目安 | 税別料金 |
|---|---:|---:|
| 会社 1 社の公開情報確認 | 10-20 req | ¥30-¥60 |
| 顧問先 100 社の月次確認 | 100-150 req/月 | ¥300-¥450/月 |
| 公開情報 DD 5 社 | 150-250 req/月 | ¥450-¥750/月 |
| BPO / 一括処理の事前確認 | 実行前に `/v1/cost/preview` で確認 | endpoint による |

実際の利用量は、検索範囲、対象数、AI agent 側の設計で変わります。広い fan-out や batch を実行する場合は、事前見積もりと上限設定を併用してください。

## 費用を制御する仕組み

| 仕組み | 用途 |
|---|---|
| `/v1/cost/preview` | 実行前に units と概算金額を確認 |
| `X-Cost-Cap-JPY` | request 単位で予算上限を設定 |
| `Idempotency-Key` | 再送時の二重実行・二重課金を避ける |
| `X-Client-Tag` | 顧問先・案件・取引先ごとの原価を分ける |
| ダッシュボード上限 | 月内の使いすぎを止める |

## 注意事項

- jpcite は一次出典付きの事実検索と scaffold を提供します。
- 税務判断、法律判断、監査意見、申請代理、許認可書面の作成は、該当する有資格者または担当者が行ってください。
- 外部 LLM の token 料金、web search 料金、cache 料金、為替、provider 側の請求仕様は jpcite が保証するものではありません。
- 料金差分の試算は、入力した前提に基づく参考値です。
