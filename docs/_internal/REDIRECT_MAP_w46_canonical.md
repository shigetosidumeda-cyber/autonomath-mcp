Decision: the brand-consolidation **intent** (new jpcite surface that
discovers / redirects to canonical content) is fully served by aliasing
the two active runbooks in `docs/_internal/`. The literal target path
`docs/canonical/` yields a no-op set, so we adapt to the actual file
location while preserving all other contract rules (banner only, source
untouched, redirect map, test coverage).

If, in a future wave, autonomath-prefixed files appear under
`docs/canonical/`, the same alias pattern applies — add jpcite-named
files alongside them with the same banner / frontmatter shape.
