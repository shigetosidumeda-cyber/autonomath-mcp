## Background

The Glue registration commit `33aca11a8` registered 40 Athena tables for Wave 69
(entity_360 × 10), Wave 70 (industry × geographic × 10), Wave 72 (AI/ML
compliance × 10), and Wave 73 (climate finance × 10). Smoke verified all
`MSCK REPAIR TABLE` succeeded, but `SELECT COUNT(*)` returned 0 for every
table because the underlying data had not been generated and synced to S3.

This task closes that gap end-to-end:

1. **Local FULL-SCALE generation** of all 40 packet kinds (`xargs -P 12` parallel)
2. **S3 sync** to `s3://jpcite-credit-993693061769-202605-derived/<prefix>/`
   (12 parallel, with sequential retry of 4 transient `SlowDown` failures)
3. **Athena COUNT(*) verification** on every packet table (10 parallel, with
   sequential retry of 10 transient `HIVE_S3_THROTTLING` failures)
4. **Crosswalk update**: 30 PENDING -> WIRED + 10 new Wave 70 entries

Per the canonical pattern (`feedback_packet_gen_runs_local_not_batch.md`,
`feedback_packet_local_gen_300x_faster.md`), generation is local Python (Python
3.12 venv) — packet build is trivial Python + SQLite, dominated by Fargate Spot
startup if attempted via Batch.
