## Lane-conflict status (read me first)

A competing concurrent agent on the same `lane:solo` claim repeatedly
reverted the H1+H2 edits during the session. The pre-commit hook then
rolled back the entire staged tree when an unrelated `mypy` failure in
`src/jpintel_mcp/mcp/autonomath_tools/answer_freshness_tool.py`
(`safe_internal_message(... code="db_open")` and a `db_open_failed`
literal that is not in the accepted error-code Literal) blocked the
commit. As a result, the final on-disk state at the end of the session
is the **prior** contract (184 manifest count + stdio+sse+streamable_http
transports), not the intended H1+H2 contract.

The remediation below records the contract decisions, the file changes
that were proposed, and the verification commands so the contract can be
re-asserted in a single follow-up commit once the lane releases and the
unrelated mypy error is fixed.

## H1 P0.1 — MCP Tool Count Contract

**Decision (D6, 2026-05-17):** MCP tool count is **219 exact**,
reconciled to live runtime cohort post Wave 51 K-S landing. The original
plan target was 218 (and an intermediate D5 land at 221). The runtime
cohort drifted slightly downward during the session as parallel Wave 51
work removed two borderline tools from the default-gate cohort; 219 is
the value that `sync_mcp_public_manifests.py` wrote into every full and
subset manifest in two consecutive runs.

### Surfaces that should hold 219 (reset by the rollback)

The script ran twice during the session and both `_meta.tool_count` and
the `packages[*]._meta...publisher-provided.tool_count` mirrors were
updated each time. The pre-commit rollback restored all of them to 184:

- `server.json`, `site/server.json`
- `mcp-server{,.full,.core,.composition}.json`
- `site/mcp-server{,.full}.json`
- `dxt/manifest.json`
- `docs/mcp-tools.md`
- `site/llms.txt`, `site/llms.en.txt`, `site/en/llms.txt`
- `site/llms-full.txt`, `site/llms-full.en.txt`
- `site/index.html` — JSON-LD MCP tools PropertyValue, 3 card body
  strings, hero count
- `site/_data/public_counts.json` — `mcp_tools_total`

### Checker contract that should be added

- `scripts/check_mcp_drift.py` — add a `MCP_TOOL_COUNT_CONTRACT = 219`
  constant; assert exact equality (not range) on every manifest and on
  the runtime probe. Leave the legacy `[lo, hi]` range as a sanity net.
- `scripts/probe_runtime_distribution.py` — flip
  `tool_count < expected_tools` (floor) to
  `tool_count != expected_tools` (exact); update the help message and
  the failure-mode hint.
- `scripts/distribution_manifest.yml` —
  `tool_count_default_gates: 184` → **219**; comment annotated with the
  H1 P0.1 contract decision; `tagline_ja: 184-tool` → **219-tool**.

## H2 P0.3 — MCP Transport Truthfulness

**Decision:** stdio is the **only** advertised MCP transport. HTTP MCP
endpoints (`/v1/mcp/sse`, `/v1/mcp/streamable_http`) are not advertised
because there is no corresponding FastAPI route mounted under
`src/jpintel_mcp/api/`. Only `/v1/mcp-server.json` (a static JSON
manifest endpoint, not a transport) is confirmed live.

### Surfaces that should be stdio-only (reset by the rollback)

- `src/jpintel_mcp/mcp/transport_metadata.py` — `MCP_TRANSPORT_NAMES`
  reduced from `("stdio", "sse", "streamable_http")` to `("stdio",)`;
  `MCP_TRANSPORT_ENDPOINTS` reduced to a single `stdio` entry;
  `MCP_TRANSPORTS_NOTE` rewritten to make the truthfulness rule
  explicit; `a2a_transport_advertisements()` docstring updated to note
  A2A skill discovery is a separate surface from MCP transport.
- All 7 primary manifests (regenerated by
  `sync_mcp_public_manifests.py`): `transports: ["stdio"]`, no `sse` /
  `streamable_http` entries, `transport_endpoints` reduced to the
  single stdio block.
- `site/.well-known/llms.json` — `mcp.endpoint` (HTTP URL) and
  `mcp.transport: "streamable_http"` replaced by
  `mcp.transport: "stdio"` + `mcp.install_command: "uvx autonomath-mcp"`
  + a `mcp.transport_note` explaining the H2 contract.
- `site/.well-known/jpcite-federation.json` — `capabilities.transport`
  reduced to `["stdio"]` with a `transport_note` block.
- `site/.well-known/agents.json` — `mcp_endpoint` (HTTP URL) replaced
  with `mcp_install_command` + `mcp_transport: "stdio"` +
  `mcp_manifest_url: "https://jpcite.com/mcp-server.json"`.
