# jpcite Cookbook v3 — コピペで即動く 10 recipe

REST (curl + jq) / Python (`requests`) / TypeScript (`fetch`) / config file (JSON / TOML) のいずれかで `curl -o` → 値を 1 箇所書き換え → 実行、で済む 10 本。各 recipe は 5 step 以内、anon (3 req/IP/日無料) でも paid (`X-API-Key`, ¥3/billable unit = 税込 ¥3.30) でも同 URL。

`X-Client-Tag` は顧問先/案件単位の課金分離 (migration 085, `usage_events.client_tag`)。`previewCost` を最初に 1 度呼べば bulk 実行前にコスト額が JSON で返る。snippet 中 `JPCITE_API_KEY` は paid 時のみ必須、anon は header ごと省略可。

## R1. 税理士事務所 — 顧問先 100 社の月次補助金スキャン

- **目的**: 巡回監査前日に「採択公表 30 日分 + 適格事業者 status + top3 候補補助金」を 100 社一括取得。
- **想定 user**: 顧問先 50-300 社の税理士事務所。
- **コピペ snippet (curl + jq, 1-line)**:
  ```bash
  jq -r '.[]' clients.json | while read HB; do \
    curl -s -H "X-API-Key: $JPCITE_API_KEY" -H "X-Client-Tag: $HB" \
      "https://api.jpcite.com/v1/corp/${HB}/360?include=adoption,invoice,enforcement,subsidy_match&months_back=1&subsidy_top_n=3" \
      | jq -c '{corp:.legal_id, invoice:.invoice_registrant.registered, new_adoption:(.adoptions|length), top:(.subsidy_matches[0].program_id), enf:(.enforcements|length)}'; \
  done | tee monthly_review_$(date +%Y-%m).jsonl
  ```
- **expected output (1 行抜粋)**:
  ```json
  {"corp":"8010001213708","invoice":true,"new_adoption":0,"top":"METI-MONOZUKURI-2026","enf":0}
  ```
- **応用 hint**: (1) `pandas.read_json("...", lines=True).to_excel(...)` で所内共有。(2) `X-Client-Tag` を弥生顧客 ID に揃え子請求書を fan-out。(3) `subsidy_top_n=10` + `prefecture` 絞りで仕分け精度を上げる。
- **コスト見積もり**: 100 社 × 1 req = ¥330 (税込) / 月。
- **失敗 pattern**: `429 too_many_requests` (anon 3 req/日超え) → `X-API-Key` 設定。`404 not_found` (法人番号 typo) → 13 桁 + チェックディジット検証で前処理。`_disclaimer.sec52` envelope は必ず転記 (税理士法 §52)。

## R2. 行政書士 — 建設業許可 prerequisite 検査

- **目的**: 法人番号 1 件入力で「必要書類リスト + 過去行政処分 history + 業法 fence」を 1 envelope。
- **想定 user**: 許認可業務を月 5-30 件回す行政書士。
- **コピペ snippet (Python, 5 行)**:
  ```python
  import os, requests, json
  HB = "8010001213708"
  r = requests.post("https://api.jpcite.com/v1/evidence/packets/query",
        headers={"X-API-Key": os.environ["JPCITE_API_KEY"], "X-Client-Tag": HB},
        json={"subject_kind":"corp","subject_id":HB,"include_facts":True,"scope":["license_construction","enforcement","invoice"]})
  print(json.dumps(r.json()["packet"], ensure_ascii=False, indent=2))
  ```
- **expected output (抜粋)**:
  ```json
  {"legal_id":"8010001213708","license_construction":{"required_docs":["登記事項証明書","納税証明書(その1,その3)","健康保険等加入証明","技術者証明"],"prereq_met":["invoice_registered","tax_paid"]},"enforcements":[],"known_gaps":["過去5年超の業務停止は corpus 未収録"],"_disclaimer":{"sec1":"本出力は提出書類チェックリストであり、許可可否判定ではありません (行政書士法§1の2・§19)。"}}
  ```
- **応用 hint**: (1) `scope` を `license_construction → license_food_service` に差し替え業種別 fence 化。(2) `subject_kind:"person"` で個人事業主の番号でも可。(3) `cron` で週次再 fetch + `corpus_snapshot_id` diff で処分新規だけ alert。
- **コスト見積もり**: 1 案件あたり 1 req = ¥3.30。
- **失敗 pattern**: `license_construction.prereq_met` が空 → `invoice_registrant` を別途 verify。`known_gaps` を無視して許可確実と誤読しない。`_disclaimer.sec1` 必須掲示。

## R3. 中小企業診断士 — 経営診断 evidence pack

- **目的**: 財務 inputs (売上 / 従業員数 / 業種 / 都道府県) → 該当補助金候補 + 業法 fence を 1 envelope。
- **想定 user**: 経営革新計画支援を月 2-10 件回す診断士。
- **コピペ snippet (curl + jq, 5 行)**:
  ```bash
  curl -s -X POST "https://api.jpcite.com/v1/programs/prescreen" \
    -H "X-API-Key: $JPCITE_API_KEY" -H "X-Client-Tag: shindan_2026Q2" \
    -H "Content-Type: application/json" \
    -d '{"revenue_jpy":350000000,"employees":42,"industry_jsic":"E","prefecture":"埼玉県","capex_plan_jpy":15000000,"top_n":5}' \
    | jq '{candidates:[.matches[]|{program_id,name,fit_score,subsidy_rate,max_amount_jpy,source_url}], known_gaps, _disclaimer}'
  ```
- **expected output (抜粋)**:
  ```json
  {"candidates":[{"program_id":"METI-MONOZUKURI-2026","name":"ものづくり補助金 第18次","fit_score":0.87,"subsidy_rate":"1/2","max_amount_jpy":12500000,"source_url":"https://portal.monodukuri-hojo.jp/..."}],"known_gaps":["市町村独自分は対象外"],"_disclaimer":{"sec1":"診断士による経営判断は別途。本出力は候補リストです。"}}
  ```
- **応用 hint**: (1) `fit_score >= 0.7` で 1 次 short-list。(2) `capex_plan_jpy` を 3 パターン投げ感応度表化。(3) 同一 input を `r4` (M&A DD) と pair で打つと売り手側の補助金履歴と買い手側候補が揃う。
- **コスト見積もり**: 1 診断 = 1 req = ¥3.30。`previewCost` 事前呼び出しで bulk 額確認。
- **失敗 pattern**: `industry_jsic` 1 文字必須 (E=製造業)。`fit_score` 計算根拠は `compatibility_rules` を展開して説明責任を保つ。

## R4. M&A advisor — DD 公開情報集約

- **目的**: 法人番号 1 件で「corp baseline + enforcement + invoice + 関連法令 fence」を 1 call。
- **想定 user**: deal volume 月 3-15 件の M&A 仲介 / FA。
- **コピペ snippet (curl + jq, 4 行)**:
  ```bash
  HB="8010001213708"
  curl -s -X POST "https://api.jpcite.com/v1/evidence/packets/query" \
    -H "X-API-Key: $JPCITE_API_KEY" -H "X-Client-Tag: deal_${HB}" -H "Content-Type: application/json" \
    -d "{\"subject_kind\":\"corp\",\"subject_id\":\"$HB\",\"include_facts\":true}" \
    | jq '.packet | {legal_id, name, adoption_count:(.adoptions|length), enforcement_count:(.enforcements|length), invoice:.invoice_registrant.registered, sources:[.sources[].url], known_gaps, _disclaimer}'
  ```
- **expected output (抜粋)**:
  ```json
  {"legal_id":"8010001213708","name":"Bookyou株式会社","adoption_count":0,"enforcement_count":0,"invoice":true,"sources":["https://www.houjin-bangou.nta.go.jp/...","https://www.invoice-kohyo.nta.go.jp/..."],"known_gaps":["採択事例 DB は中企庁公表 2,286 件のみ"],"_disclaimer":{"sec72":"本出力は DD checklist であり、法律判断ではありません。"}}
  ```
- **応用 hint**: (1) `corpus_snapshot_id` を deal memo に転記し締結時点の差分監査 baseline 化。(2) 同 envelope を seller / buyer 双方で取り突合表に。(3) `enforcement_count > 0` 案件は `r11` enforcement 全文に drill-down。
- **コスト見積もり**: 1 target = 1 req = ¥3.30 (個別 endpoint なら 3-4 call, 同金額)。
- **失敗 pattern**: 非公開 / 未登記法人は `404`。海外子会社 (法人番号無し) は対象外。`_disclaimer.sec72` を deal memo に必ず転記。

## R5. 信用金庫渉外 — 顧客 watch (月次 amendment + 制度 candidate)

- **目的**: 渉外担当の顧客 list (法人番号) → 月次「補助金新規採択 + 行政処分 + 制度 candidate」一斉 push。
- **想定 user**: 信用金庫 / 商工会の渉外担当 (顧客 30-150 社)。
- **コピペ snippet (Python, 5 行)**:
  ```python
  import os, requests, csv
  for hb in [r["houjin_bangou"] for r in csv.DictReader(open("customers.csv"))]:
      r = requests.get(f"https://api.jpcite.com/v1/corp/{hb}/360", params={"include":"adoption,enforcement,subsidy_match","months_back":1,"subsidy_top_n":3},
            headers={"X-API-Key": os.environ["JPCITE_API_KEY"], "X-Client-Tag": f"branch_omiya/{hb}"})
      d = r.json(); print(f"{hb}\t{len(d['adoptions'])}\t{len(d['enforcements'])}\t{(d['subsidy_matches'] or [{}])[0].get('program_id','-')}")
  ```
- **expected output (TSV 抜粋)**:
  ```
  8010001213708	0	0	METI-MONOZUKURI-2026
  7010001234567	1	0	SAITAMA-DX-2026
  ```
- **応用 hint**: (1) `branch_omiya/...` 形式 tag で支店別月次レポート。(2) 採択検知 (`adoption>0`) は融資 cross-sell trigger に。(3) `r2` 行政処分 watch と pair で reputation risk monitor。
- **コスト見積もり**: 100 社 × 月 1 回 = ¥330 (税込) / 月 / 支店。
- **失敗 pattern**: `X-Client-Tag` 重複時は最後の tag で集計、支店別請求書発行は `tag` 命名規約 (`branch_xxx/houjin`) を統一。`429` は anon 残数で発生 → key 必須。

## R9. REST direct integration — anonymous 試走から paid 移行

- **目的**: LLM SDK を使わず純 REST で jpcite を叩き、独自 RAG pipeline に embed。
- **想定 user**: 既存 RAG / agent framework (LangChain / LlamaIndex 等) を持つ agent dev。
- **コピペ snippet (TypeScript, 5 行)**:
  ```ts
  const r = await fetch("https://api.jpcite.com/v1/programs/search?q=" + encodeURIComponent("DX 補助金") + "&prefecture=東京都&tier=S,A&limit=10",
    { headers: { "X-API-Key": process.env.JPCITE_API_KEY ?? "", "X-Client-Tag": "rag_pipeline_v2" } });
  const d = await r.json();
  for (const p of d.matches) console.log(p.program_id, p.tier, p.source_url);
  ```
  Python 等価 (`requests.get(...).json()`) も同 URL / 同 header で動く。LLM 推論は呼び出し側で実装。
- **expected output**:
  ```
  METI-MONOZUKURI-2026 S https://portal.monodukuri-hojo.jp/...
  TOKYO-DX-2026 A https://www.metro.tokyo.lg.jp/...
  ```
- **応用 hint**: (1) anon (header 省略) で 3 req/日試走 → 採用後に `X-API-Key` 追加で本番化、URL は不変。(2) `corpus_snapshot_id` を vector DB のメタに保存し monthly diff invalidation。(3) `previewCost` を 1 度叩いて bulk 額を JSON で取得してから運用判断。
- **コスト見積もり**: 1 req = ¥3.30。10,000 req/月 = ¥33,000 (税込)。
- **失敗 pattern**: `401` (key 形式 NG) / `429` (anon 残 0 or rate burst) / `422` (param validation)。`Retry-After` header に従い backoff。

## R10. Bash one-liner — 公的制度 sanity check 集

- **目的**: 終端で 1 行コピペ → 値 1 つ書き換え → JSON 抜粋、で済む確認集。
- **想定 user**: 全 cohort 共通 (ad-hoc 確認 / CI smoke / Slack bot bash)。
- **コピペ snippet (curl + jq, 5 件)**:
  ```bash
  # 1) 法人番号で 360 view
  curl -s -H "X-API-Key: $JPCITE_API_KEY" "https://api.jpcite.com/v1/corp/8010001213708/360" | jq '.legal_id, .invoice_registrant.registered'

  # 2) 補助金キーワード検索 (anon 可)
  curl -s "https://api.jpcite.com/v1/programs/search?q=ものづくり&prefecture=東京都&limit=5" | jq '.matches[] | {program_id, tier, source_url}'

  # 3) 適格事業者番号 bulk verify (T + 13 桁)
  for T in T8010001213708 T7010001234567; do curl -s -H "X-API-Key: $JPCITE_API_KEY" "https://api.jpcite.com/v1/invoice_registrants/search?invoice_registration_number=$T" | jq -c "{t:\"$T\", ok:.matches[0].registered}"; done

  # 4) cost 事前見積 (bulk 前に 1 度)
  curl -s -X POST -H "X-API-Key: $JPCITE_API_KEY" -H "Content-Type: application/json" -d '{"endpoint":"/v1/corp/{id}/360","calls":100}' "https://api.jpcite.com/v1/billing/preview" | jq '.total_jpy_incl_tax'

  # 5) 行政処分 30 日分
  SINCE=$(date -v -30d +%Y-%m-%d); curl -s -H "X-API-Key: $JPCITE_API_KEY" "https://api.jpcite.com/v1/enforcement-cases/search?event_type=clawback&event_date_after=$SINCE&limit=50" | jq '.matches[] | {case_id, recipient_houjin_bangou, event_date, source_url}'
  ```
- **expected output (4 番)**:
  ```json
  330
  ```
- **応用 hint**: (1) 1) を `xargs -P 8` で 8 並列。(2) 4) を CI ステップ最上段に置き bulk 暴走を防ぐ。(3) 5) を cron + `tee enforcement_$(date +%F).jsonl` で日次 archive、`unified_id` で前日差分 diff。
- **コスト見積もり**: 1) 3) 5) = ¥3.30/req、2) 4) = anon 可 = ¥0。
- **失敗 pattern**: `jq` 未 install (macOS: `brew install jq`)。日付関数は GNU `date` と BSD `date` で書式違い、上記は BSD (macOS) 表記。
