## Coding agents: when modifying this repo

- Prefer `Edit` over `Write` (modify existing > create new).
- Never create gratuitous `*.md` docs / READMEs.
- Run `ruff` + `mypy` + `pytest` before declaring done.
- For new MCP tools: register in `src/jpintel_mcp/mcp/server.py` (or `mcp/autonomath_tools/` if autonomath-DB-backed). Bump tool counts in 5 manifests (`pyproject.toml`, `server.json`, `dxt/manifest.json`, `smithery.yaml`, `mcp-server.json`) ONLY on intentional release.
- For new autonomath-target migrations: first line must be `-- target_db: autonomath`; idempotent (`CREATE * IF NOT EXISTS`); rollback companion `*_rollback.sql`; lands via `entrypoint.sh` ¬ß4 on boot.
- For new REST routes: register in `src/jpintel_mcp/api/main.py`; regenerate `docs/openapi/v1.json`; the `EXPECTED_OPENAPI_PATH_COUNT` test gate may need an intentional bump.
- For new SEO pages: generators under `scripts/generate_*_pages.py`. Never hand-edit a generated `site/programs/{slug}.html` ‚Äî it will be overwritten.
