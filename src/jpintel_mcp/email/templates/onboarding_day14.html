1. MCP 経由でループを LLM に押し込む

curl で 1 件ずつ叩いている場合、Claude Desktop / Cursor / Cline から MCP 経由で呼ぶと、自然言語側でクエリ生成と結果の正規化がそのまま噛みます。下の設定を 1 回入れれば、以降は「東京都の中小企業向けで 1000 万円以上の補助金を10件、それぞれ排他ルールも付けて」と書くだけです。

{
  "mcpServers": {
    "autonomath": {
      "command": "uvx",
      "args": ["autonomath-mcp"],
      "env": { "JPCITE_API_KEY": "jc_xxx" }
    }
  }
}

クライアント別セットアップ詳細

2. /programs/batch で N 倍の節約

取引先の DD / 申請候補のスクリーニングなど、N 件まとめて取りに行くなら 1 リクエストにまとめるのが効きます。N 個の get_program を 1 回の POST にできて、課金も 1 回分です。

curl -X POST -H "X-API-Key: jc_xxx" -H "Content-Type: application/json" \
  "https://api.jpcite.com/v1/programs/batch" \
  -d '{"unified_ids":["UNI-14e57fbf79","UNI-40bc849d45","UNI-08d8284aae"]}'

取引先の法人番号 N 件は /v1/am/dd_batch にまとめられます。50 件まで 1 リクエスト・¥3.30 で、補助金採択 / 行政処分 / 適格請求書発行登録を一括確認できます。

3. monthly_cap_yen で予算上限を縛る

従量課金の不安は 「気づいたら ¥10 万」に集中します。鍵単位の月次 ¥ ハード上限を 1 行で設定できます(JST 月初 00:00 リセット)。例えば月 ¥3,000 まで:

curl -X POST -H "X-API-Key: jc_xxx" -H "Content-Type: application/json" \
  "https://api.jpcite.com/v1/me/cap" \
  -d '{"monthly_cap_yen": 3000}'

上限到達後はそのキーが 402 Payment Required を返します。本番アプリでも安心して使えるようになります。解除は {"monthly_cap_yen": null} です。
