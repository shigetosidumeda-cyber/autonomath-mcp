Cursor MCP に jpcite を 30 秒で接続

curl -o .cursor/mcp.json https://jpcite.com/.cursor/mcp.example.json 1 行で 155 MCP 機能を利用できます。Cursor 0.42+ 必須、anon free 3 req/IP/日。

5 分接続手順

  1. Cursor 0.42 以降を確認: Cursor → Settings → About。0.41 以下は MCP 未対応のため要更新。
  2. uvx を準備: curl -LsSf https://astral.sh/uv/install.sh | sh でインストール (既存ユーザはスキップ)。
  3. プロジェクト root に .cursor/mcp.json を配置: mkdir -p .cursor && curl -o .cursor/mcp.json https://jpcite.com/.cursor/mcp.example.json
  4. Cursor を再起動: Settings → MCP を開き、jpcite が緑 dot で出ているか確認。
  5. (任意) API key を埋め込み: env.JPCITE_API_KEY を jc_... に置換 (新規は jc_ プレフィクス、既発行の sk_ / am_ も互換)。anon free のままなら省略可。

copy-paste snippet

.cursor/mcp.json をプロジェクト root に作成。

{
  "mcpServers": {
    "jpcite": {
      "command": "uvx",
      "args": ["autonomath-mcp"],
      "env": {
        "JPCITE_API_KEY": "jc_xxxxx"
      }
    }
  }
}

グローバルに設定する場合は ~/.cursor/mcp.json に同一内容を配置。

動作確認

Cursor のチャットで以下を入力。jpcite が呼ばれ、source_url 付きの結果が返れば成功。

@jpcite search_programs q="ものづくり補助金" prefecture="東京都"

Settings → MCP → jpcite の右側に tools count: 151 が表示されれば接続成功です。36協定書面の生成機能は公開提供していません。

REST 直叩きで疎通確認する場合は -G --data-urlencode を必ず使うこと。日本語 query を URL に直接埋めると HTTP リクエスト行が壊れ、API は {"error":{"code":"bad_request",...,"hint":"Use --data-urlencode for non-ASCII query params..."}} を返します。

curl -G "https://api.jpcite.com/v1/programs/search" \
  --data-urlencode "q=ものづくり補助金" \
  --data-urlencode "prefecture=東京都" \
  -H "X-API-Key: ${JPCITE_API_KEY:-}"

FAQ

anonymous と paid の違いは?

anon は IP ベースで 3 req/日 (AnonIpLimit)、API key 無し。paid は税別 ¥3/billable unit metered で上限なし、JPCITE_API_KEY 経由でカウント。

API key はどう発行する?

/pricing.html#api-paid から Stripe Checkout (consent は custom_text.submit.message) で発行。直後にダッシュボードで jc_... を表示 (新規は jc_ プレフィクス、既発行の sk_ / am_ も互換)。

8 業法 fence とは?

jpcite は税理士法§52・弁護士法§72 等の独占業務に抵触する個別助言を返しません。/legal-fence.html 参照。
