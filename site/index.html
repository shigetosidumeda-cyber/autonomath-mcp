制度プレスクリーン — 3 問で 上位 5 件

都道府県・事業形態・予定投資額を入れるだけで、11,684 制度から適合度の高い上位 5 件を表示します。無料・登録不要・匿名 50 リクエスト/月/IP。 入力 1 回で結果が返ります。

「全国」を選ぶと地域条件を外して全制度から検索します。都道府県を選ぶと、その県専用制度＋全国制度の両方が候補になります。
「指定なし」だと両方を含む制度のみヒット。「法人」「個人事業主」を選ぶと専用制度＋共通制度を表示。
設備・システム導入などの予定投資額（万円単位、空欄可）。例: 800 = 800 万円。投資額に応じた上限額の制度を上位に並べます。

回答は AI ではなく、制度データベースから規則ベースでマッチングします (prefecture + target_types + amount_fit)。出典 URL は一次資料のみ。送信内容は保存しません。レート制限のため IP ハッシュのみ記録 (JST 月初リセット)。

ChatGPT に聞けば済む話では?

短答: いいえ。3 つの理由で 事実水準 の制度判定には使えません。

学習日が古い (知識の打ち切り日問題)

時間軸が合わない

ChatGPT / Claude の学習データは数ヶ月〜1 年前で固定です。補助金は「公募締切=昨日」のような時間軸で動くため、AI 単独では「今申請できるか」が判断できません。jpcite は全行に更新日を持ちます。

出典の捏造 (幻覚リスク)

誤判定 = 受給機会の損失

AI は「存在しない補助金 ID」や「実在しない URL」を生成する事例が報告されています。¥300 万〜¥3,000 万規模の誤判定は申請失敗・依頼者トラブルの主因です。jpcite は全 14,472 件に出典 URL と更新日を付与、一次資料のみに限定しています (景表法対応)。

構造化されていない (併用可否を機械判定できない)

併用可否は推論で決まらない

AI は「補助金 A と B の併用可否」や「同一事業者の他案件との重複」を構造的に判定できません。jpcite は 181 本の 併用不可 / 前提制度 ルールで併用可否を機械判定します。

つまり: AI + jpcite = 速度 (AI) × 正確性 (一次資料 + ルール判定)。jpcite は AI の代替ではなく、AI が信頼できる出力を出すための事実層。

なぜ jpcite か

集約サイトを採用しない

情報精度の確保

noukaweb・補助金ポータル・biz.stayway 等の二次情報源には、「掲載中の制度が既に終了」「上限金額が古い」「存在しない要件を追記」といった事例が複数報告されています。これを根拠に申請して不採択となれば、事務所への信頼喪失や苦情に直結します。jpcite はこれらを出典に登録しません (禁止)。出典は省庁 (METI / MAFF / 中企庁)・都道府県・政府系金融 (JFC / SMRJ / NEDO)・商工会の一次 URL のみ。14,472 件の 99%以上で一次 URL を保有し、残る 12 件 (小規模自治体) は「一次 URL 不足」と明示します。全行が出典 URL と更新日を持つため、「なぜこの補助金がヒットしたか」を顧客・監督官庁・訴訟相手に客観的に提示できる監査ログを生成できます。詳細は 透明性レポート を参照。

一次資料 100%

毎日 定期処理 で出典 URL の生存確認

全制度に出典 URL と更新日を付与。 出典更新日は透明に開示します (「最終更新」のような誤誘導はしません)。 毎晩、出典 URL の死活監視を実施し、 法改正の差分も日次で走査する仕組みを実装済みです (差分の蓄積は初回フル走査後)。
