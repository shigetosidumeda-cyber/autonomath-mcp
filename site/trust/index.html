jpcite は AI が日本の公的根拠を扱う際の first hop として 設計されている。下表は、その契約 (envelope contract / disclaimer floor / anchor seal / acceptance contract) が 外向け claim と一致しているかを 7 行で開示する。すべての行は monitoring/* または analytics/* の JSONL に anchor され、ファイル名で再現可能。

SEED と OBSERVED の分離: 各セルに real_count と synth_count を併記し、 seed estimate を observation として読まれないようにする 。
Proof asset Status Last refreshed Verified count (real) Synth count Anchor file

各行の意味

JCRB benchmark
4-arm benchmark。analytics/jcrb_summary_latest.json の real_calls_total と synthesized_count を分けて表示。 seed estimate (kind="seed") が混入していたら row は red。
Practitioner eval
15 persona × 3 query = 45 row の合格率。 must_include 100% × must_not_claim 0 × boundary 維持。 site/practitioner-eval/_data/results_latest.json 経由。
Composite benchmark
artifact_id ベースの応答品質スコア。 monitoring/composite_benchmark_latest.jsonl の real_calls_total / synthesized_count を分離表示。
Source receipts coverage
有料応答のうち _source_receipts が空でない比率。 monitoring/source_receipts_coverage.jsonl 経由。 閾値 99% (green) / 95% (yellow)。
Known gaps display rate
応答に _known_gaps が現れた比率。 monitoring/known_gaps_display.jsonl 経由。 閾値 95% (green) / 85% (yellow)。
Anchor verify success rate
audit_seal の verify roundtrip 成功率 (発行 → 取得 → 復号 → 検証)。 monitoring/audit_seal_roundtrip.jsonl 経由。 閾値 99% (green) / 95% (yellow)。
Acceptance contract pass rate
有料 artifact の最低契約 (envelope keys + disclaimer + source_url + source_fetched_at) を満たした応答比率。 monitoring/acceptance_contract_pass.jsonl 経由。

免責

本 page は jpcite が自社サービスを自分で計測した結果。 第三者監査ではない。各行のソース JSONL は repo に常駐し、reproducible に 検証できる。 Public claim (例: トップページの「source_receipts 100%」) を 出すときは、その数字に対応する Trust Center 行を 必ず引用すること。 CI: tests/test_landing_claims_in_trust_center.py がブロックする。
