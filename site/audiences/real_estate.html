なぜ 不動産業 利用者層 pack か

1 リクエスト で全部

補助金 + 通達 + 裁決事例

従来は search_programs + find_saiketsu + cite_tsutatsu を別々に呼ぶ必要があった。pack_real_estate は 1 リクエスト に集約し、課金も 1 件 (¥3 税別)。

不動産業 fence 自動適用

JSIC K + 名称キーワード

「不動産・空き家・住宅・賃貸・改修・流通・リフォーム・耐震・省エネ住宅・既存住宅」のキーワード fence を自動適用。JSIC D の住宅政策系も自動取り込み。

所得税・相続税系 重視

所基通 + 相基通

不動産業特有の所得税 (居住用財産譲渡所得・住宅借入金特別控除) ・相続税 (小規模宅地等の特例・相続時精算課税) 関連の通達・裁決事例を優先返却。

3 つの代表クエリ

1. 「大阪府 不動産業 10人 補助金」

# MCP (Claude Desktop / Cursor)
pack_real_estate(
  prefecture="大阪府",
  employee_count=10,
  revenue_yen=200_000_000
)

# REST
curl -H "X-API-Key: sk_..." \
  "https://api.zeimu-kaikei.ai/v1/am/pack_real_estate?prefecture=大阪府&employee_count=10"

1 リクエスト ¥3。top 10 補助金 + 5 裁決事例 + 3 通達。

2. 「居住用財産 譲渡所得」

# 国税不服審判所 直接検索
find_saiketsu(
  query="居住用財産 譲渡",
  tax_type="所得税"
)

pack に含まれない裁決事例を補完。

3. 「小規模宅地等の特例」

# 通達 直接検索
cite_tsutatsu(
  code="相基通-69-4-1"
)

# または措置法を直接
search_tax_incentives(
  q="小規模宅地"
)

相続税系の通達・税制措置を直接参照。

5 分で開始

  1. API キー発行 — メアド登録のみ、カード登録は 50 リクエスト/月 free を超えてから。
  2. Claude Desktop / Cursor 設定: uvx autonomath-mcp。詳細は developer 向け。
  3. 「pack_real_estate」を呼ぶ — pref / 従業員数 / 年商を渡すだけで top 10 補助金 + 通達 + 裁決事例。
  4. 結果の _next_calls に従って search_programs / find_saiketsu へ深掘り。

免責事項 (税理士法 §52 / 公認会計士法 §47条の2)

本ページおよび pack_real_estate tool の出力は、jpintel programs (不動産業 fence) + nta_saiketsu + nta_tsutatsu_index 一次資料の機械的 aggregation です。 税務助言 (税理士法 §52) ・監査調書 (公認会計士法 §47条の2) ・申請代理 (行政書士法 §1) ・経営判断 (中小企業診断士の経営助言) の代替ではありません。 業種マッピングは JSIC K + 名称キーワード fence による heuristic で、各 program の適合可否は申請要領を一次資料 (source_url) で必ずご確認ください。 裁決事例・通達は公表時点の解釈であり、改正により現在の取扱が変更されている可能性があります。 相続税・所得税の個別判断 (居住用財産譲渡・小規模宅地等の特例・住宅借入金特別控除等) は資格を有する税理士・公認会計士に必ずご相談ください。
