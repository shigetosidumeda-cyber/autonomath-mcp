不動産業特化 jpcite

不動産業特化の補助金候補抽出 + 国税庁通達 (所得税基本通達 / 相続税基本通達) + 採択事例を 1 API で。不動産業コンサル / 全国宅地建物取引業協会連合会 (全宅連) 担当者 / 不動産専門 SaaS 連携開発者向け。エンドユーザー (宅建業者・不動産投資家 個人) ではなく、その上流に立つ「経由チャネル」を主対象に再設計しています。

対象

  • 不動産業特化 補助金コンサル / 中小企業診断士 / 認定支援機関 — 顧問先宅建業者・賃貸事業者の補助金候補と通達根拠を 1 リクエストでまとめて返す
  • 全国宅地建物取引業協会連合会 (全宅連) / 全日本不動産協会 / 不動産協会 担当者 — 会員企業向けの補助金 widget や会報誌コンテンツを jpcite から自動更新
  • 不動産専門 SaaS / 賃貸管理 SaaS / 仲介支援ツール 連携開発者 — 物件管理・契約管理アプリに「使える補助金候補」プラグインを差し込む

主要価値

  • JSIC K fence (不動産業) — 不動産・空き家・住宅・賃貸・改修・流通 キーワード union で 189 件 の S/A/B/C tier program を絞り込み (snapshot 2026-04-30 時点)
  • 所基通 (所得税基本通達) + 相基通 (相続税基本通達) のセクション固定参照 — 居住用財産譲渡・住宅借入金特別控除・小規模宅地等の特例 等のセクション ID 付き引用
  • 国土交通省 行政処分 cohort — 宅建業法違反等の処分 79 件 (ministry='国土交通省', 建設業と corpus 共有) を houjin_bangou で横断検索、与信審査・反社チェックの補助情報として併記可
  • JSIC K 採択事例 11 件 (K 親 2 / K68 不動産取引業 1 / K69 不動産賃貸業・管理業 4 / K70 物品賃貸業 4) — honest 注記: 不動産業の採択事例は補助金性質上 極端に薄い corpus です。「使える数」というより「同業の採択がそもそも少ない」という事実情報として併記する想定。提案書添付時はこの薄さを前提に。
  • 各補助金候補・各通達・各裁決事例・各処分・各採択事例に 一次資料 source_url を併記 (官庁・自治体・公庫などの一次資料に限定)
  • 1 リクエスト ¥3 (税込 ¥3.30) 完全従量、3 リクエスト/日まで無料 (匿名 IP ベース、JST 翌日 00:00 リセット)

経由チャネルでの使い方

  • コンサル登録 — 認定支援機関の方へ。顧問先不動産業者の補助金候補を pack_real_estate 1 コールでまとめて取得 → 提案書に所基通 / 相基通 セクション ID 付きで添付。outreach 用の API 利用説明・ドキュメント直リンクは下部 CTA から。
  • 業種団体 widget 設置 — 全宅連 / 全日本不動産協会 / 不動産協会 の会員サイト / 会報誌 Web 版に、jpcite の補助金候補 widget を埋込。会員企業からは協会経由で見える形。
  • 不動産専門 SaaS integration 開発 — 物件管理・賃貸管理・契約管理アプリで pack_real_estate を呼び出し、物件登録時に「使える補助金候補」を併記。実装例は integration guide に掲載。

API 実例

curl "https://api.jpcite.com/v1/am/pack_real_estate?prefecture=神奈川県&employee_count=12" \
  -H "X-API-Key: $JPCITE_API_KEY"

corpus 開示 (2026-04-30 snapshot): JSIC K 関連 program 189 件 / JSIC K 採択事例 11 件のみ (薄い corpus) / 国土交通省 行政処分 79 件 (建設業と共有)。採択事例の数が少ないため、提案書では補助金候補と通達引用が主軸となります。本サービスは情報検索です。相続税・所得税の個別判断は資格を有する税理士・公認会計士にご相談ください (税理士法 §52 / 公認会計士法 §47条の2 / 宅地建物取引業法 §3)。本ページは LLM による生成・要約は行っておらず、SQLite + 一次資料 URL のみで構成されています。

API キー発行 → コンサル登録 (認定支援機関 outreach) → 業種団体 widget 設置 (全宅連 / 不動産協会経由) → 不動産専門 SaaS integration 開発 → 料金
