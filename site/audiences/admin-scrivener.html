許認可案件 1 件 = 補助金 + 融資 + 行政処分 + 採択事例 を 3-5 query で抽出 = ¥9-15。 100 案件/月でも ¥1,500。 地域 × 業種 × 締切で絞込、 排他ルール 181 本で併用失格を機械検出。

行政書士の代替ではありません

税務会計AI は申請書類を作成しません。許認可申請の代理は行政書士法 §1-3 により行政書士の独占業務であり、税務会計AI が代行することはありません。税務会計AI は 事前リサーチ (どの許認可がこの顧客に必要か / 排他ルール / 過去の処分事例) のみを提供します。許認可申請の業務量を減らすのではなく、リサーチ工数を減らして 1 案件あたりの粗利を上げるためのツールです。

なぜ 税務会計AI か

1 リクエスト で 4 データセット横断

programs + loans + enforcement + cases

許認可案件で「使える補助金 + 融資 + 行政処分歴 + 採択事例」を 1 リクエスト で取得。Claude / GPT に投げて提案書ドラフトまで自動化できる。

排他ルール 181 本

併用失格を事前検出

建設業向けの「ものづくり補助金 + 事業再構築補助金」のような併用可否を 181 本のルールで判定。「気づかないまま併用申請 → 後日失格」を回避。

地域 × 業種 × 締切

期限間近案件の発掘

都道府県 × JSIC 業種 × 締切日でフィルタ。「東京都 × 建設業 × 30 日以内」で当月案件を抽出、提案アジェンダに組込み。

主要 use case

1. 建設業許可更新 + 補助金提案

許可更新で訪問する案件で、target_types=「建設業」+ prefecture を 1 リクエスト。ものづくり補助金・建設キャリアアップシステム関連・各県建設業向け制度を抽出し、更新書類とセットで提案。

2. 介護事業者の指定申請 + 関連助成

介護保険法上の指定申請を扱う際、介護職員処遇改善加算・特定処遇改善加算・ICT 導入支援補助金を関連 program として抽出。労働環境整備の助成も付帯。

3. 運送業 (一般貨物自動車運送) 開業支援

運送業許可申請に合わせ、トラックドライバー労働環境整備支援・働き方改革推進支援補助金・JFC 運送業向け融資を 1 リクエスト で取得。融資は担保・保証人三軸で分解済。

4. 産廃処理業の許可 + 行政処分歴

産業廃棄物処理業の許可案件で、競合・取引先の行政処分歴 (廃棄物処理法・建設業法) を法人番号で確認。enforcement_cases 1,185 件から該当を抽出。

5. 認定支援機関連携 (経営革新計画)

経営革新計画・先端設備等導入計画の認定取得を支援する際、認定取得後に使える補助金・税制優遇を抽出。事業再構築・ものづくり補助金の認定要件と紐付け。

よくある質問 (行政書士)

建設業許可の更新案件で補助金は探せますか?

はい。target_types=「建設業」+ prefecture を絞ると、ものづくり補助金・事業再構築補助金・各都道府県建設業向け制度等が抽出されます。期限間近順でソート可。

介護・運送・産廃の許認可とどう紐付けますか?

業種コード (JSIC) と target_types で絞込。介護事業者向け補助金、運送業向け労働環境整備補助金、産廃処理業向け省力化補助金などが該当します。

排他ルール 181 本とは何ですか?

「補助金 A と補助金 B は併用不可」「補助金 C は事前に認定 D が必要」等の関係を構造化した 181 ルール (125 exclude + 17 prerequisite + 15 absolute + 24 other)。program_ids を投げると衝突を返します。

案件 1 件あたりの req 数は?

事前リサーチ 5-10 リクエスト + 提案書作成 5-10 リクエスト → 案件 1 件 10-20 リクエスト 目安。月 30 件処理で 300-600 リクエスト → ¥900-1,800 (税別)。

認定支援機関の登録案件にも使えますか?

はい。ものづくり補助金・事業再構築補助金は認定支援機関の関与が要件。税務会計AI で関連制度を抽出し、士業 affiliate で 専門家 登録すれば 中小企業 からの問合せも受けられます。

行政処分歴を確認できますか?

はい。enforcement_cases 1,185 件 (建設業法・宅建業法・産廃処理法等) を法人番号・名称で照会可能。VC / M&A 専門家 領域とも重なります。

関連 利用者層 は?

税理士 (顧問先共有案件)、中小企業経営者 (相談元)、VC / M&A 専門家 (DD enforcement check)、AI 開発者 (事務所内 RAG 自作) と use case が重なる。5 利用者層 全体を比較。

行政書士向けリソース

  • 中小企業 経営者ページ (相談元の理解) — 顧客側がどんな目線で 税務会計AI を使っているかを把握。LINE bot からの紹介経由で 専門家 登録 = 安定した相談元になる。
  • REST API: /v1/programs/search — 許認可案件の関連補助金を 1 リクエスト で抽出。target_types + prefecture + industry で絞込、排他ルール 181 本で併用失格を事前検出。
  • REST API: /v1/enforcement-cases/search (1,185 件) — 建設業法・宅建業法・産廃処理法等の行政処分歴を法人番号・名称で照会。許認可承継 DD でも使える。
  • 料金 (¥3/リクエスト 完全従量) — 案件 1 件 10-20 リクエスト 目安、月 30 件処理で ¥900-1,800 (税別)。月額固定 0 円、最低利用なし。
  • 事務所サイトに埋め込む (widget) — 自事務所 LP に 事前確認 を貼って相談接点を作る。1 行 script タグで動く、white-label OK。
  • 士業 affiliate に登録 — LINE bot 経由で 中小企業 から相談を受ける側になる。登録無料、1 紹介成立 ¥3,000 (税別) 専門家 受取。地域 × 業種 × 専門分野で profile 公開。
  • 5 分で開始 (クイックスタート) — API キー発行から最初の検索までの最短ルート。Free 50 リクエスト/月で 5-10 顧客分は完全無料運用可能。

採択率公表値の透明性

採択率公表値: 69 制度 / 10,790 中 (0.64%)

公的機関が round ごとの採択率を明示公表している制度は 10,790 中わずか 69 制度 (0.64%) です。 残り 99.36% は 「採択率公表データなし」 と honest に明示し、 推測値は出しません。 この 0.64% カバレッジを 隠さず明示 するのは、 営業現場で 「採択率○%」 を盛って提示することが誤情報リスクと隣り合わせだからです。

採択率データが取得できる主な制度の例 (2026-04-25 時点): ものづくり補助金 65 round (mean 48.29%)、 事業再構築補助金 48 round (mean 39.55%)、 小規模事業者持続化補助金 46 round (mean 54.88%)、 IT導入補助金 31 round (mean 64.54%)。 残り 10,721 制度の per-program ページには 「採択率公表データなし」 とのみ表示します。
