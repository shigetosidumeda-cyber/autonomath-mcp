利用者層ごとに返すもの

jpcite は長い資料を毎回 AI に詰め込む使い方ではなく、AI が次の回答や業務メモに貼りやすい「根拠付きの小さな成果物」を返します。価格差はその後ろにある補足で、まず見るべきなのは成果物の型です。

Evidence Packet

制度ID・キーワード・法令名から、一次資料 URL、取得時刻、known gaps、引用候補、業法 fence note、token 見積もりを返します。

Evidence Packet を見る

会社フォルダ

法人番号・T番号・会社名から、folder_readme、CRMメモ、owner_questions、source_receipts、known_gaps、recommended_followup_by_channel を返します。Brief preview は 1 unit、Pack workflow は複数 section の見積もり units で課金されます。

顧問先月次レビュー

顧問先の所在地・業種・会社名をもとに、候補制度、締切、変更点、顧問先に聞く質問を定例確認用にまとめます。

baseline 注記: この表は「どの利用者が、どの Evidence Packet / Brief / 監視出力を AI に渡せるか」を見るための対応表です。費用は req 数 × ¥3 税別 の固定式で、外部サービス費、削減額、売上、利益、専門判断の価値は保証しません。token + web search の API 料金だけのコスト比較は Evidence cost calculator で前提を明示して確認してください。

提供感: 長い資料を毎回 AI に渡す前に、source_receipts、known_gaps、確認質問、引用候補を小さな packet として返します。ユーザーが買うのは「節約額」ではなく、根拠付きの次アクションを短い文脈で AI に渡せることです。
