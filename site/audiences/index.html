関連性 (cross-reference)

audience は重なる。立場の組合せに応じて使い分ける。

税理士 ↔ 行政書士

顧問先紹介で連携するケース。税理士は措置法・確定申告に強く、行政書士は許認可・補助金書類に強い。税理士 ・ 行政書士 両 page で重複 use case を整理した。

SMB → 士業 affiliate

SMB 経営者は LINE bot で発見し、必要なら士業 affiliate (advisor 登録) に繋ぐ。advisor は登録無料・成約時のみ ¥3,000/件。SMB → advisor 探索。

VC ↔ AI developer

DD pipeline は社内 RAG として組み立てるケース多。VC が要件定義し、developer が MCP / REST で実装。VC + developer page を組合せる。

税理士 → AI developer

事務所内 RAG を構築する税理士は、MCP server を Claude Desktop / Cursor に直結。税理士 → developer page。
