関連性 (cross-reference)

利用者層 は重なる。立場の組合せに応じて使い分ける。

税理士 ↔ 行政書士

顧問先紹介で連携するケース。税理士は措置法・確定申告に強く、行政書士は許認可・補助金書類に強い。税理士 ・ 行政書士 両 page で重複 use case を整理した。

行政書士 ↔ 補助金コンサルタント

補助金 専門家 の 9 割は中小企業診断士・FP・無資格コンサル・認定支援機関、 行政書士は少数派です。 申請書類の代理は行政書士独占ですが、 制度の事前リサーチは士業独占ではないため、 多くの 専門家 は 補助金コンサル page が該当します。 行政書士は 行政書士 page (許認可案件) を併読してください。

中小企業 → 士業 affiliate

中小企業 経営者は LINE bot で発見し、必要なら士業 affiliate (専門家 登録) に繋ぐ。専門家 は登録無料・成約時のみ ¥3,000/件。中小企業 → 専門家 探索。

VC ↔ AI developer

DD 処理経路 は社内 RAG として組み立てるケース多。VC が要件定義し、developer が MCP / REST で実装。VC + developer page を組合せる。

税理士 → AI developer

事務所内 RAG を構築する税理士は、MCP サーバ を Claude Desktop / Cursor に直結。税理士 → developer page。
