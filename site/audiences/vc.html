なぜ AutonoMath か

法人番号 1 query で 4 dataset 横断

enforcement + cases + invoice + programs

13 桁の法人番号 (国税庁公表) を投げるだけで、行政処分・採択履歴・適格事業者登録・関連制度を統一スキーマで返却。社内 DD ツールに REST or MCP で統合。

一次資料のみ、出典 URL 必須

aggregator 不採用

行政処分は各監督官庁公表分、採択は経産省・農水省・中小企業庁・各自治体公表分、適格事業者は国税庁 PDL v1.0。aggregator (二次情報源) は使わない。各行に source_url + fetched_at 付き。

scale 制限なし、¥3/req 一律

月額固定・seat なし

月 1M req でも単価は ¥3 (税込 ¥3.30) のまま。最低金額・年間契約・seat fee なし。Stripe メータード請求で運用 scalable、適格請求書 (T8010001213708) 自動発行。

なぜ VC がこれを使うべきか — IRR の話

  • ポートフォリオ会社が取りこぼしている補助金 = 事業外収益。100社 × 平均1件 × 平均¥500万 = ¥5億の取りこぼし。1年で IRR を 1-2pt 改善する原資。
  • DD で「未受給の優良制度」を発見 = 投資後 100 日プランの quick win。経営チームの信頼獲得に直結。
  • 行政処分・ネガ判明 = 投資判断の事前回避。1案件あたり数千万〜数億の毀損回避はファンド全体の IRR を守る。

主要 use case

1. 投資候補の compliance check

シードラウンドの 50 社を法人番号 batch 投入 → 行政処分歴 (建設業法・金商法・電気通信事業法等)、適格請求書登録の有無、関連補助金採択歴を一覧化。投資判断会議の事前資料を自動生成。

2. M&A buy-side DD: 過去 5 年の行政処分

買収対象 + 主要取引先 30 社の法人番号で行政処分歴を 1 batch。check_enforcement_am tool で重大処分 (営業停止・登録取消) を即抽出、レポート添付。

3. 採択歴で技術検証

NEDO・経産省ものづくり補助金・中小企業庁事業再構築の採択履歴は技術力の客観指標。search_acceptance_stats_am で業種別採択率 + 当該社の採択回数を取得。

4. ポートフォリオ会社の支援可能制度

投資後の value-up 文脈で、ポートフォリオ会社が使える補助金・税制特例・融資を 1 call で抽出。排他ルール 181 本で「これとこれは併用不可」を事前検出。

5. インボイス対応 DD

取引先・仕入先の適格請求書発行事業者登録を法人番号で確認。invoice_registrants 13,801 件 (PDL v1.0 attribution、月次フル更新準備中) で登録日・業態を照会。

Python batch (商用 ¥3/req)

# pip install httpx
import httpx, asyncio

# DD 標準パッケージ: 法人番号 1 件で 3 endpoint を chain、複数法人を並列実行
nums = ["8010001213708", "1010001067912"]  # 13 桁の国税庁発番号
BASE = "https://api.autonomath.ai/v1"

async def dd(client, num):
    invoice = await client.get(f"{BASE}/invoice_registrants/T{num}")
    enforce = await client.get(f"{BASE}/enforcement-cases/search",
                               params={"corporate_number": num,
                                       "since": "2021-01-01"})
    cases = await client.get(f"{BASE}/case-studies/search",
                             params={"q": num, "limit": 20})
    return {"invoice": invoice.json(), "enforcement": enforce.json(),
            "case_studies": cases.json()}

async def main():
    async with httpx.AsyncClient(headers={"X-API-Key": "sk_..."}) as client:
        results = await asyncio.gather(*[dd(client, n) for n in nums])
    for r in results:
        print(r)

asyncio.run(main())

よくある質問 (VC / M&A advisor)

DD pipeline で 100 社並列に投げられますか?

はい。¥3/req metered で hard cap なし、スパイク時も 429 になりません。100 社 × 5 endpoint = 500 req → ¥1,500 (税別) で 1 batch 完了します。

行政処分歴は何年遡れますか?

enforcement_cases 1,185 件 (建設業法・宅建業法・産廃処理法・金商法・電気通信事業法等)。原則 5 年遡及、自治体公開分は遡及期間が異なる場合あり。各行に source_url + fetched_at 付き。

採択歴はどう取得しますか?

case_studies 2,286 件 (補助金採択公表分) を法人番号・名称で照会。10 年程度遡及可能 (制度ごとに公表期間が異なる)。search_acceptance_stats_am で業種・地域別の採択率も確認できます。

適格請求書発行事業者の確認は?

invoice_registrants 13,801 件 delta-only (PDL v1.0 attribution、国税庁公表 bulk pending)。法人番号で登録有無 + 登録日 + 業態を照会。インボイス対応の DD 観点で必須。

scale 制限はありますか?

enterprise scale でも料金は ¥3/req 一律 metered。月額固定・seat fee・年間最低なし。月 1M req でも単価は変わりません。Stripe メータード請求で運用 scalable。

PII / 取引履歴は含まれますか?

AutonoMath は公的開示された一次資料 (行政処分公表・採択結果公表・適格事業者登録 PDL v1.0) のみを扱います。個人 PII・取引履歴・財務情報は含みません。商業 DB との組合せ運用が前提です。

DD レポート自動化に組み込めますか?

はい。MCP server を Claude / GPT に直結 + REST endpoint を社内 RAG に統合する 2 系統。dd_profile_am tool が法人番号 1 件で標準 DD パッケージを返します。

関連 audience は?

投資先 / 売主側で重なる stakeholders: 税理士 (税務 DD)、行政書士 (許認可承継 DD)、AI agent 開発者 (社内 DD bot を組む側)、SMB 経営者 (投資先候補)。5 audience 全体を比較。
