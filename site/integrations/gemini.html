Integrations / Gemini

Gemini で 税務会計AI を使う

5 分で動く統合ガイド (登録不要 / Free 50 req/月)

所要時間: 5 分以内。必要なもの: Gemini CLI (Node 18+) または Gemini Extensions (Workspace) + Python 3.10 以降。API キー不要 — 50 req/月 per IP は匿名で動きます。stdio 接続、protocol 2025-06-18 対応。
2 つの経路:
  • Gemini CLI (推奨): MCP stdio ネイティブサポート。~/.gemini/settings.json に JSON 1 行で 89 tools フル接続。本ページの主経路。
  • Gemini Extensions (Workspace): MCP は未対応のため OpenAPI を OpenAPI Tool として登録。REST 経由で同じデータソース (11,684 件のプログラム + 9,484 法令 + 13,801 適格事業者) にアクセス。

Step 1 — API キーを取得 (任意)

50 req/月 までは認証不要 (anonymous tier、IP 単位、JST 月初 00:00 リセット)。本番運用は Dashboard から API キー (sk_...) を発行 → Stripe metered で ¥3/req 自動課金 (税込 ¥3.30、最低料金・年契約・tier なし)。signup form は トップページ または Dashboard から。

Step 2 — autonomath-mcp + Gemini CLI をインストール

autonomath-mcp:

pip install autonomath-mcp

または uvx ephemeral 実行 (PATH 汚染なし、自動アップデート):

uvx autonomath-mcp

Gemini CLI:

npm install -g @google/gemini-cli

動作確認:

autonomath-mcp --version
# → autonomath-mcp 0.3.1
gemini --version
# → @google/gemini-cli x.y.z

Step 3 — ~/.gemini/settings.json を編集

OS ごとの設定ファイルパス:

OS settings.json
macOS / Linux~/.gemini/settings.json
Windows%USERPROFILE%\.gemini\settings.json

ファイルが無ければ新規作成:

{
  "mcpServers": {
    "zeimu-kaikei": {
      "command": "autonomath-mcp"
    }
  }
}
API キー入りバージョン (50 req/月 を超える運用) 
{
  "mcpServers": {
    "zeimu-kaikei": {
      "command": "autonomath-mcp",
      "env": {
        "ZEIMU_KAIKEI_API_KEY": "sk_xxxxxxxxxxxx"
      }
    }
  }
}
uvx 版 (PATH 解決の trouble を回避) 
{
  "mcpServers": {
    "zeimu-kaikei": {
      "command": "uvx",
      "args": ["autonomath-mcp"]
    }
  }
}

Gemini Extensions (Workspace) の場合

Gemini Extensions は MCP を直接話せないため、OpenAPI を Extensions Tool として登録します:

  • aistudio.google.com → Build → Extension
  • OpenAPI URL: https://api.zeimu-kaikei.ai/v1/openapi.json
  • Authentication: API Key (header X-API-Key) または None (anonymous)

Step 5 — Troubleshoot

1. command not found: autonomath-mcp

  • which autonomath-mcp でフルパス取得 → "command": "/Users/you/.local/bin/autonomath-mcp" のように絶対パスで指定
  • あるいは "command": "uvx", "args": ["autonomath-mcp"] に切り替え

2. Gemini CLI が MCP サーバを起動しない

  • gemini --debug で起動ログ確認 (stderr に Python トレースバックが出れば pip 再インストール)
  • JSON が valid か: python -m json.tool ~/.gemini/settings.json

3. 50 req/月 の残量を確認

  • Gemini に get_usage_status を呼んで残クォータを教えて と指示。このツール呼び出し自体はクォータを消費しません (probe-only)
  • JST 月初 00:00 に自動リセット
