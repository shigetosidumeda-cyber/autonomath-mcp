Integrations / Continue.dev

Continue.dev で 税務会計AI を使う

5 分で動く統合ガイド (登録不要 / Free 50 req/月)

所要時間: 5 分以内。必要なもの: Continue.dev 拡張 (VS Code 1.85+ または JetBrains 2024.2+) + Python 3.10 以降 (pip または uvx)。API キー不要 — 50 req/月 per IP は匿名で動きます。stdio 接続、protocol 2025-06-18 対応。
ボリューム: プログラム 11,684 件 (検索可) / 全 14,472 件 / 89 MCP tools / autonomath-mcp v0.3.1。Continue は MCP の stdio をネイティブサポートするので 1 つの YAML で 89 tools 全公開。

Step 1 — API キーを取得 (任意)

50 req/月 までは認証不要 (anonymous tier、IP 単位、JST 月初 00:00 リセット)。本番運用は Dashboard から API キー (sk_...) を発行 → Stripe metered で ¥3/req 自動課金 (税込 ¥3.30、最低料金・年契約・tier なし)。signup form は トップページ または Dashboard。

Step 2 — autonomath-mcp + Continue 拡張をインストール

Continue 拡張をインストール:

  • VS Code: Extensions タブ (Cmd+Shift+X / Ctrl+Shift+X) で Continue.continue を検索 → Install
  • JetBrains: Settings → Plugins → Marketplace で Continue を検索 → Install

autonomath-mcp をインストール:

pip install autonomath-mcp

または uvx で ephemeral 実行 (PATH 汚染なし、自動アップデート):

uvx autonomath-mcp

動作確認:

autonomath-mcp --version
# → autonomath-mcp 0.3.1

Step 3 — ~/.continue/config.yaml を編集

OS ごとの設定ファイルパス:

OS config.yaml
macOS / Linux~/.continue/config.yaml
Windows%USERPROFILE%\.continue\config.yaml

VS Code から開く: コマンドパレット (Cmd+Shift+P) → Continue: Open config.yaml。Project-level に書きたいときは .continue/config.yaml をプロジェクトルートに作成。

以下を mcpServers セクションとして追加 (既存があれば zeimu-kaikei エントリだけ追記):

mcpServers:
  - name: zeimu-kaikei
    command: autonomath-mcp
    args: []
API キー入りバージョン (50 req/月 を超える運用) 
mcpServers:
  - name: zeimu-kaikei
    command: autonomath-mcp
    args: []
    env:
      ZEIMU_KAIKEI_API_KEY: sk_xxxxxxxxxxxx
uvx 版 (PATH 解決の trouble を回避) 
mcpServers:
  - name: zeimu-kaikei
    command: uvx
    args: ["autonomath-mcp"]

YAML 保存後、Continue が自動で MCP サーバを再起動。サイドバーの Continue ペイン → MCP セクションで zeimu-kaikei (89 tools) と表示されれば成功。

Step 4 — 動作確認 + 期待される response

Continue chat (Cmd+L / Ctrl+L) を開いて以下を貼り付け:

search_programs を呼んで prefecture=東京都 + target_type=中小企業 + keyword=省エネ で 3 件取得。primary source URL もつけて。

Continue が search_programs ツール呼び出しの承認ダイアログを出します (read-only ツールは autoApprove 設定で承認スキップ可)。

Step 5 — Troubleshoot

1. MCP セクションに表示されない / "Failed to start"

  • Continue ペイン → 該当行 → Show Logs でスタックトレース確認
  • 大抵は autonomath-mcp: command not found → which autonomath-mcp 結果を絶対パスで指定
  • あるいは command: uvx + args: ["autonomath-mcp"] に切替

2. YAML syntax error

  • YAML はインデントとリストが厳しい。python -c "import yaml; print(yaml.safe_load(open('~/.continue/config.yaml')))" で valid 確認
  • VS Code の YAML 拡張で開けば赤線で indicate されます

3. 50 req/月 の残量を確認

  • Continue chat に get_usage_status を呼んで残クォータを教えて と指示。このツール呼び出し自体はクォータを消費しません (probe-only)
  • JST 月初 00:00 に自動リセット
