OpenAI Custom GPT で 税務会計AI を使う

5 分で動く統合ガイド (登録不要 / Free 50 リクエスト/月)

所要時間: 5 分以内。必要なもの: ChatGPT Plus / Team / 大規模利用 (Custom GPT 作成権限)。API キー不要 — anonymous tier 50 リクエスト/月 per IP は認証なしで動きます。本番運用は API キーを Bearer で。
仕組み: Custom GPT は MCP プロトコルを直接話せませんが、OpenAPI 3.x スペックを Import すると Actions として 111 エンドポイントを呼べます。MCP の 89 ツールと同じデータソース (補助金 / 法令 / 適格事業者 / 採択事例 / 税制ルール / 行政処分 等) に REST 経由でアクセス。

Step 1 — Custom GPT を新規作成

  1. ChatGPT (chatgpt.com) を開いてログイン
  2. 左サイドバー Explore GPTs → + Create (右上)
  3. または直接 chatgpt.com/gpts/editor を開く
  4. 上部タブを Configure に切り替え (Create タブのウィザードはスキップ)

Step 2 — Name + Description + Instructions を設定

以下を貼り付け (任意調整可):

Name:

日本制度サーチ (税務会計AI)

Description:

日本の補助金・融資・税制・認定・採択事例・行政処分・法令・判例・入札・適格事業者を一次資料 URL 付きで横断検索する AI アシスタント。

Instructions:

あなたは日本の公的制度 (補助金・融資・税制優遇・認定制度) の検索アシスタントです。

ルール:
1. 必ず Actions の 税務会計AI API を呼んでから回答する。記憶や訓練データから補助金額・締切を答えない。
2. 各回答に必ず source_url + fetched_at (出典取得日) を併記する。
3. 個別の税務・法務助言は税理士法 §52 により禁止。「上記は情報検索結果です。具体的な助成判断は税理士・行政書士へ」と末尾に必ず付ける。
4. 検索のヒット件数が 0 のときは『該当なし』と正直に答える。類似制度を勝手にでっち上げない。
5. 適格請求書発行事業者 (T-番号) の登録状況は search_invoice_registrants で必ず実検索。
6. 排他ルール (補助金併用可否) は check_exclusions で必ず判定。回答内に断定的に『併用可』と書く前に必ず実行。

レスポンス形式:
- search 系は {total, limit, offset, results[]} 形式
- 各 row は source_url + source_fetched_at を持つ → これを引用形式で示す
- _disclaimer フィールドが付いていたらそのまま末尾に表示

Step 3 — Actions に OpenAPI を Import

Configure タブの一番下に Actions セクション。+ Create new action をクリック。

Action 編集画面の 構造 セクションで Import from URL をクリックし、以下を貼り付け:

https://api.zeimu-kaikei.ai/v1/openapi.json

Import がうまく動かない場合は手動で URL を fetch してテキストを直接 構造 欄に貼り付け (curl: curl -s https://api.zeimu-kaikei.ai/v1/openapi.json > openapi.json)。

Import 成功時、Available actions に operation_id 一覧 (search_programs / get_program / search_invoice_registrants / search_laws ... 計 111) が表示されます。

Step 4 — Authentication を設定

Option A — Anonymous (50 リクエスト/月 per IP, 認証なし)

  • Authentication 欄を None に設定
  • 50 リクエスト/月 制限は OpenAI 側 IP に紐づくため、Custom GPT を使うユーザー全員で 1 つの quota を共有することに注意

Option B — APIキー (本番運用、¥3/リクエスト 完全従量)

  • Dashboard で APIキー (sk_...) を発行
  • Authentication → APIキー を選択
  • Auth Type: Bearer
  • APIキー: sk_xxxxxxxxxxxx をそのまま貼り付け (Bookyou 側で X-API-Key 形式も Bearer 形式も両方受理)

Step 5 — Privacy policy を貼って保存

OpenAI は Custom GPT の公開時に Privacy policy URL を要求します:

https://zeimu-kaikei.ai/privacy.html

右上の Save から公開範囲を選択 (Only me / Anyone with link / Public)。Test 欄で動作確認すれば完了。

最初のプロンプト

Custom GPT のプレビュー or 公開後 GPT を開いて貼り付け:

中小企業向けの省エネ補助金を3件探して。primary source URL もつけて。

ChatGPT が search_programs Action を呼ぶ承認ダイアログを出します (初回のみ)。Always Allow をクリックすると以後自動で呼びます。

Troubleshooting

1. "Could not find a valid URL in servers" エラー

  • OpenAPI スペックの servers[].url は絶対 URL である必要があります。https://api.zeimu-kaikei.ai を使用 (相対パスの /v1 は不可)
  • Bookyou 提供の /v1/openapi.json は仕様準拠なので Import 失敗時はブラウザキャッシュをクリアしてリトライ

2. Action 呼び出しで 401 Unauthorized

  • APIキー (sk_...) を Bearer で送るとき、頭の Bearer プレフィックスは ChatGPT が自動で付与。キー本体だけを貼る
  • Anonymous tier で動かしているのに 401 が出るときは、ChatGPT 側が前回のキャッシュを送っている。Action を一度削除して再 Import

3. Action 呼び出しで 429 Rate limited

  • Anonymous tier (50 リクエスト/月) は OpenAI 側の単一 IP に紐づき、Custom GPT 利用者全員で共有。多人数公開する GPT は API キーで運用 (¥3/リクエスト)
  • JST 月初 00:00 リセット

4. ChatGPT が Action を呼ばずに training data から答えてしまう

  • Instructions 冒頭の "必ず Actions の 税務会計AI API を呼んでから回答する" の文言を強化
  • "検索結果なしで答えるな" / "If you cannot call the API, respond '検索失敗' and stop" を追記

5. 日本語パラメータが文字化け / 0 件返る

  • API は UTF-8 + URL-encode を内部で処理。Custom GPT 側は OpenAPI スキーマに従って自動でエンコード
  • 0 件のときは enum_values エンドポイントで実在する値 (例: target_type の正規語彙) を確認するよう Instructions に追記
