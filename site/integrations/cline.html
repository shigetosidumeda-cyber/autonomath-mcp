Integrations / Cline (VS Code)

Cline (VS Code) で 税務会計AI を使う

5 分で動く統合ガイド (登録不要 / Free 50 リクエスト/月)

所要時間: 5 分以内。必要なもの: VS Code 1.85 以降 + Cline 拡張 (または Roo Code) + Python 3.10 以降 (pip または uvx)。API キー不要 — 50 リクエスト/月 per IP は匿名で動きます。

Step 1 — Cline 拡張をインストール

VS Code を起動し、Extensions タブ (Cmd+Shift+X / Ctrl+Shift+X) で Cline を検索:

  • 正式パッケージ ID: saoudrizwan.claude-dev (publisher: Cline)
  • 類似の RooVeterinaryInc.roo-cline (Roo Code) も同じ MCP 設定形式で動作

Install をクリック。インストール後、左サイドバーに Cline ロボットアイコンが追加されます。

Step 2 — autonomath-mcp をインストール

VS Code 内蔵ターミナル (Ctrl+`) で:

pip install autonomath-mcp

動作確認:

autonomath-mcp --version
# → autonomath-mcp 0.3.1

Step 3 — Cline の MCP Servers タブを開く

  1. 左サイドバー Cline アイコンをクリック
  2. Cline ペインの上部メニューから MCP Servers タブ (サーバアイコン)
  3. Configure MCP Servers をクリック → cline_mcp_settings.json が VS Code で開きます

設定ファイルの実体パス (参考):

OS cline_mcp_settings.json
macOS~/Library/Application Support/Code/User/globalStorage/saoudrizwan.claude-dev/settings/cline_mcp_settings.json
Windows%APPDATA%\Code\User\globalStorage\saoudrizwan.claude-dev\settings\cline_mcp_settings.json
Linux~/.config/Code/User/globalStorage/saoudrizwan.claude-dev/settings/cline_mcp_settings.json

Step 4 — JSON 設定を貼り付け

開いた cline_mcp_settings.json に以下を追記 (既存の mcpServers オブジェクトがあれば "zeimu-kaikei" キーだけ追加):

{
  "mcpServers": {
    "zeimu-kaikei": {
      "command": "autonomath-mcp",
      "disabled": false,
      "autoApprove": []
    }
  }
}

autoApprove について: Cline は MCP ツール呼び出しごとに承認ダイアログを出します。autoApprove にツール名を列挙すると承認スキップ。read-only ツールだけを auto-approve するのが安全 (例: ["search_programs", "get_program", "search_laws", "get_meta", "enum_values", "get_usage_status"])。書き込み系は将来追加されないため自動承認しても安全ですが、初回は手動承認推奨。

API キー入りバージョン (50 リクエスト/月 を超える運用) 
{
  "mcpServers": {
    "zeimu-kaikei": {
      "command": "autonomath-mcp",
      "disabled": false,
      "autoApprove": [],
      "env": {
        "ZEIMU_KAIKEI_API_KEY": "sk_xxxxxxxxxxxx"
      }
    }
  }
}

API キーは Dashboard で発行 (¥3/リクエスト 完全従量, 税込 ¥3.30)。

uvx 版 (PATH 解決のトラブルを回避) 
{
  "mcpServers": {
    "zeimu-kaikei": {
      "command": "uvx",
      "args": ["autonomath-mcp"],
      "disabled": false,
      "autoApprove": []
    }
  }
}

Step 5 — 設定を保存して試す

JSON を保存 (Cmd+S / Ctrl+S)。Cline が自動で MCP サーバを再起動し、MCP Servers タブの zeimu-kaikei 行に緑のドット + 89 tools 表示。

Cline チャットを開いて貼り付け:

zeimu-kaikei MCPで、中小企業向けの省エネ補助金を3件探して。primary source URL もつけて。

Cline が search_programs ツール呼び出しの承認ダイアログを出します。Approve をクリックすると JSON レスポンス (補助金 3 件 + source_url) が表示され、Cline がそれを整形して返答。

出力サンプル

[Tool] zeimu-kaikei → search_programs
  args: {"keyword": "省エネ", "target_type": "中小企業", "limit": 3}
  → 200 OK (3 results, 0.34s)

Cline で活きるプロンプト

  • このプロジェクトの README を読んで、対象の事業に合う補助金を prescreen_programs で評価 — Cline が現在ファイルを読んで profile 化
  • 請求書 CSV の取引先 法人番号 を全件 search_invoice_registrants で検証して登録切れを抽出 — Cline がループで MCP 呼び出し
  • 事業再構築 + ものづくり 同時申請可否を check_exclusions で判定
  • 製造業×埼玉県 SMB スターターパック (補助金 + 融資 + 採択事例 + 締切) → smb_starter_pack

Troubleshooting

1. MCP Servers タブにサーバが表示されない

  • JSON が valid か確認: python -m json.tool ~/Library/Application\ Support/Code/User/globalStorage/saoudrizwan.claude-dev/settings/cline_mcp_settings.json
  • 余分なカンマ / クォート抜けで詰まりがち
  • VS Code を再起動 (Developer: Reload Window コマンド)

2. 赤いドット ("Failed to start")

  • Cline ペインの MCP Servers → 該当行 → Show Logs でスタックトレース確認
  • 大抵は autonomath-mcp: command not found → which autonomath-mcp 結果を絶対パスで指定
  • あるいは "command": "uvx", "args": ["autonomath-mcp"] に変更

3. ツール呼び出しの承認ダイアログが煩わしい

  • autoApprove に頻用 read-only ツールを列挙: ["search_programs", "get_program", "search_laws", "search_invoice_registrants", "get_meta", "enum_values", "get_usage_status"]
  • 本 API は読み取り専用 (書き込み系ツールなし) なので副作用なし

4. Windows: 日本語レスポンスの文字化け

  • "env": {"PYTHONIOENCODING": "utf-8"} を追加

5. 50 リクエスト/月 の残量を確認

  • Cline に get_usage_status を呼んで残クォータを教えて — このツール呼び出し自体はクォータを消費しません (probe-only)
  • JST 月初 00:00 に自動リセット
