jpcite vs japan-corporate-mcp (yamariki-hub): 機能・データ範囲・併用パターン (2026)

yamariki-hub/japan-corporate-mcp は gBizINFO + EDINET + e-Stat の live API proxy として動作する OSS MCP wrapper です。jpcite は 166,969 法人 + 13,801 適格事業者 + 1,185 行政処分 を pre-indexed で返す 151 tool MCP。jpcite は analyst pre-screening、japan-corporate-mcp は live regulator pulls (API key を既に provisioning 済の場合) で住み分け。

本ページは公開情報をもとに当社が作成しています。 数値は 2026-05-04 時点で japan-corporate-mcp GitHub README + pyproject.toml および PyPI から取得しました。最新情報は各 OSS リポジトリと当社 料金ページ をご確認ください。 誤りに気づいた場合は info@bookyou.net までご連絡いただければ 48 時間以内に修正します。

当社を選ぶケース / japan-corporate-mcp を選ぶケース

jpcite が向いている

  • analyst pre-screening: 法人番号 1 本で 適格請求書 + 行政処分 + 補助金受給歴 + 採択事例 + 関連法令 を 1 envelope で取得し スクリーニングしたい。japan-corporate-mcp は 4 tool 連続呼び出しが必要。
  • API key プロビジョニング不要: 匿名 3 req/日 で out-of-the-box 試行可能。japan-corporate-mcp は gBizINFO + EDINET + e-Stat 3 個の登録 (EDINET は MFA) が前提。
  • 適格請求書 13,801 + monthly 4M bulk + 行政処分 1,185 + structured enforcement detail を法人と join したい (japan-corporate-mcp 対象外)。
  • audit-grade Evidence Packet (corpus_snapshot_id + _disclaimer envelope) を auditor / DD レポートに添付したい。

japan-corporate-mcp が向いている

  • live regulator pulls (keys are already provisioned): gBizINFO + EDINET + e-Stat の API key を既に provisioning 済で、上場企業の有価証券報告書を即時取得したい (jpcite は snapshot ingest)。
  • e-Stat 任意統計の direct query: 国勢調査・経済センサス等 e-Stat の任意 dataset を ad-hoc に query したい (jpcite は ingest 済 aggregates のみ)。
  • EDINET 即時 disclosure 対応が業務要件 (jpcite は cache 由来でラグあり)。

推奨併用パターン

役割が違う: japan-corporate-mcp は 3 regulator API の live proxy (key provisioning 済で上場企業 disclosure / 任意 e-Stat query)、jpcite は 法人 + 適格 + 行政処分 + 補助金 を pre-indexed で 1 envelope。競合ではなく complement。

1. M&A / DD の最初のスクリーニングは jpcite の graph_traverse + match_due_diligence_questions + cross_check_jurisdiction で 法人番号 360° (登記 + 適格 + 行政処分 + 補助金受給歴 + 関連 採択) を 1 call 取得。

2. 上場企業ターゲットで EDINET 有価証券報告書の即時詳細が必要な場合のみ japan-corporate-mcp の get_edinet_reports を ad-hoc に呼ぶ (EDINET key は user 側 provisioning)。

3. 任意統計 (例: 業種別事業所数) の direct query が必要な場合のみ get_statistics を呼ぶ。jpcite の ingest 済 aggregates で足りる場合は jpcite で完結。

4. 監査人 / 弁護士 提出時は jpcite の corpus_snapshot_id + _disclaimer envelope で再現性を担保し、必要に応じて regulator API URL を併記。

注意点

  • japan-corporate-mcp は gBizINFO / EDINET / e-Stat の3 個別登録が必要。EDINET は MFA 必須で onboarding に数日。
  • 各 regulator の API quota は user 持ち。大量バッチクエリは quota 超過リスクあり。
  • 公式 MCP registry には未登録 (jpcite は io.github.shigetosidumeda-cyber/autonomath-mcp として登録済)。
