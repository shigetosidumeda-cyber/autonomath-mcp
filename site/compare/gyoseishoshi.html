LLM 単独経路の隠れコスト

  • Source hallucination — 出典 URL を捏造するリスク. jpcite は一次資料 URL + fetched_at で構造的に防ぐ.
  • known_gaps 概念なし — LLM は「分からない」を返さず憶測する. jpcite は欠損 surface を artifact 化.
  • aggregator 汚染 — LLM の training set には noukaweb / hojyokin-portal 等が含まれる. jpcite は aggregator ban.
  • 士業独占業務の越権リスク — LLM は §52 / §47条の2 / §72 / §1 / §3 を境界として明示しない. jpcite は disclaimer envelope 必須.
  • fetched_at の semantic honesty — LLM は「最新」と言うが何時取得かは保証しない. jpcite は「出典取得 YYYY-MM-DD HH:MM」を必ず返す.
