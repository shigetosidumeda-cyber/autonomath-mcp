jpcite vs jgrants-mcp-server (デジタル庁 OSS MCP): 機能・データ範囲・併用パターン (2026)

digital-go-jp/jgrants-mcp-server はデジタル庁が公開する jGrants 公式 API の OSS MCP wrapper です。jpcite は 11,684 制度 + 9,484 法令 + 2,286 採択事例 + 13,801 適格事業者 + 22,258 enforcement detail + 50 税制 + 2,065 判例 を横断する 93 tool MCP。両者は競合ではなく住み分け — jgrants-mcp は申請ポータル直結、jpcite は適合判定 + 横断引用。

本ページは公開情報をもとに当社が作成しています。 数値は 2026-05-04 時点で jgrants-mcp-server GitHub README + src/jgrants_mcp_server/core.py および 公式 MCP registry から取得しました。最新情報は各 OSS リポジトリと当社 料金ページ をご確認ください。 誤りに気づいた場合は info@bookyou.net までご連絡いただければ 48 時間以内に修正します。

当社を選ぶケース / jgrants-mcp を選ぶケース

jpcite が向いている

  • cross-source compliance check: 補助金 + 融資 + 税制 + 認定 + 法令 + 判例 + 行政処分 を 1 回の MCP call で 横断引用したい (jgrants-mcp は jGrants 補助金のみ)。
  • 排他 / 併用 ルール 181 件 をプログラム同士の matrix で取得し 併用判定したい。
  • audit-grade Evidence Packet (corpus_snapshot_id + _disclaimer envelope) を auditor に渡したい。
  • 法人番号 → 適格請求書 → 行政処分 → 補助金受給歴 を 1 call で 360° に まとめたい。

jgrants-mcp-server が向いている

  • grant application path: jGrants gBizID 電子申請までの最短経路を取りたい (jpcite は検索・適合判定までで、申請は jGrants 経由を推奨)。
  • デジタル庁公式の OSS であることが法務・調達上の前提 (Copyright Digital Agency, Government of Japan)。
  • 添付資料 PDF / Word / Excel の Markdown 化 が主目的 (get_file_content 専用、jpcite には対応 tool なし)。

推奨併用パターン

役割が完全に違う: jgrants-mcp は jGrants 公式 API 直結 (申請動線)、jpcite は 11+ 一次資料の横断 (適合判定動線)。競合ではなく complement で、典型ワークフローは以下:

1. jpcite で list_open_programs + search_acceptance_stats_am + apply_eligibility_chain_am により事業者プロフィールに合う open 公募を抽出 + 採択事例 + 排他/併用 + 関連 行政処分 を 1 envelope で取得。

2. 該当した jGrants 制度 ID を jgrants-mcp の get_subsidy_detail + get_file_content に渡し、申請要領 + 公募要領 PDF を Markdown で取得。

3. jpcite の get_law_article_am + cite_tsutatsu で 関連 法令 / 通達 / 直近改正を 引用補強。

4. 申請は jgrants-mcp 経由 (= jGrants 公式 API の電子申請 path) で実施。

注意点

  • jgrants-mcp-server は PyPI 配布なし。git clone + uv venv での自前 install が必要。
  • 主要 LLM client (Claude Desktop / Cursor / Roo-Code) との接続は FastMCP CLI proxy 経由が想定。
  • 公式 MCP registry には未登録 (jpcite は io.github.shigetosidumeda-cyber/autonomath-mcp として登録済)。
