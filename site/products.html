成果物カタログ

jpcite は「検索API」ではなく、AI が回答を書く前に必要な根拠だけを小さな Evidence Packet / output artifact として返すレイヤーです。長い PDF・公式ページ・検索結果を毎回 LLM に読ませる代わりに、一次資料 URL、取得時刻、known gaps、引用候補を保存・共有・監査できる形にまとめます。

接続・インストールは Claude / Cursor / ChatGPT / Codex 向け接続ガイド、API キー発行は Pricing、ブラウザ確認は Playground から始められます。

成果物カード

Evidence Packet

制度ID・キーワード・法令名を入れると、長い PDF や公式ページを毎回 LLM に読ませる代わりに、一次資料URL、取得時刻、known gaps、業法 fence、引用候補を小さな Evidence Packet として返します。社内レビューや AI agent の回答生成前に使えます。

匿名 3 req 枠でプレビュー →

顧問先月次レビュー

顧問先名・所在地・業種を入れると、税制・補助金・融資候補、出典URL、未確認点、顧問先へ送る質問文を月次レビュー用に返します。事務所の定例確認で使う根拠収集・未確認点整理のための出力です。返るのは根拠・未確認点・質問文であり、業務時間や成果は利用者の運用に依存します。

保存検索を見る →

会社フォルダ Brief / Pack

法人番号・T番号・会社名から、folder_readme、CRMメモ、owner_questions、source_receipts、known_gaps、recommended_followup_by_channel を会社フォルダへ貼りやすい形に整理します。Brief preview は 1 unit、複数 section を束ねる Pack workflow は見積もり上の req 数で課金されます。保存・監視は利用者側の設定に依存します。

会社フォルダをプレビュー →

申請前 Evidence Packet

地域・業種・投資額を入れると、候補制度、除外条件、併用注意、一次資料 URL、known gaps、次に聞く質問を申請前の打合せ資料として返します。申請書作成・提出判断ではなく、一次資料確認と候補整理に課金されます。

匿名 3 req 枠でプレビュー →

公開情報 DD

取引先名・法人番号を入れると、公開情報、行政処分、採択履歴、未確認範囲をDDメモとして返します。融資・M&A・取材前の裏取り担当向けです。

DDの始め方 →

併用・排他チェック表

候補制度の ID リストを入れると、併用不可、前提制度不足、unknown、確認すべき一次資料を表で返します。AI に推論させず、申請前の確認対象を分けられます。

API reference →

インボイス取引先確認表

T番号・法人番号・取引先 CSV から、登録状態、法人同定、変更履歴、再確認が必要な行を返します。経理・税理士レビューの前処理に使えます。

取引先確認の始め方 →

Source Receipt

source URL・制度ID・法令IDごとに、取得時刻、content hash、known gaps、再確認対象をまとめます。社内説明や顧問先メモの「どこまで確認したか」に使えます。

出典を見る →

法令・判例引用候補

法令名・条文・制度・争点キーワードから、引用候補、関連条文、関連判例、専門家確認の注意を返します。AI 回答やレビュー資料の根拠候補になります。

法令カタログ →

改正・保存検索通知

監視キーワード・法令・制度を登録すると、改正差分、影響候補、通知ログ、Webhook delivery を受け取れます。メール・RSS・Webhook で保存検索の変化を追えます。

通知管理を見る →

費用・支払い制御

API key・Stripe・Wallet・x402 payment proof をもとに、利用量、上限、二重課金防止、支払い検証、再実行可否を返します。Wallet は無料クレジットではなく、agent が実行前に「この call は支払い可能か、上限内か、再送で二重課金されないか」を確認する payment-control output です。

Agent handoff

Claude・Cursor・ChatGPT・Codex へ同じ evidence を渡せるように、REST、MCP、OpenAPI surface をそろえて返します。人と agent の間で根拠の取り違えを減らします。
