cite chain の組成方法

1 つの programs 行は、複数の一次資料を順序付けた cite chain として保持されます。例えば「ものづくり補助金」(UNI-PROG-monozukuri-...) の cite chain は次のように構成されます:

1 中小企業庁 (発出機関)

経済産業省の外局である中小企業庁を発出機関として記録。am_authority table に entity 化。
role=primary_source

2 公募要領 PDF (一次資料)

中小企業庁公開の公募要領 PDF (例: https://www.chusho.meti.go.jp/keiei/...)。
am_source.is_pdf=1 · license=gov_standard_v2.0 · fetched_at=2026-04-15

3 事務局公式サイト (申請窓口)

中小企業庁が指定する事務局の公開ページを併記 (申請窓口情報)。
role=application_url · license=gov_standard_v2.0

4 採択事例 (LINK・参考)

関連する採択事例を関係性 link edge で結ぶ。
role=reference

cite chain の各 step は、am_entity_source (entity ↔ source の many-to-many 結合) と am_source (URL・取得日時・ライセンス・PDF flag を保持) の JOIN で組成されます。検索結果や MCP ツール 戻り値の source_url は、この chain の primary step の URL です。

API 経由で cite chain を取得する

個別レコードの cite chain は GET /v1/am/programs/{program_id}/sources で取得できます。public API 経路・auth 不要。

$ curl 'https://api.zeimu-kaikei.ai/v1/am/programs/UNI-PROG-monozukuri-xxx/sources'

個別 record の出典が古い・リンク切れ・原典齟齬を発見された場合は、info@bookyou.net まで連絡ください。dispute (利用者からの指摘) として一次資料と再突合し、48 時間以内 (営業日) に判断します。

集約サイト (二次情報サイト) を採用しない方針

税務会計AI は noukaweb / hojyokin-portal / biz.stayway 等の二次情報サイトを source_url に採用しません。

これは利用規約レベル (CLAUDE.md データ衛生規定) ではなく、コード側の 除外リスト でも強制されています。

過去に二次情報サイトを出典とすると、原典との乖離 (古い金額・既に終了した制度・存在しない要件) をそのまま AI に渡してしまう事故がありました。利用者から見れば「税務会計AI が嘘をついた」になり、最悪の場合は受給見込み違いや不適切申請につながり、これは重大な誤情報リスクです。

そのため当社は、source_url に集約系の URL を入れない方針を以下の 3 層で固めています:

  • 規定層 — CLAUDE.md "Data hygiene" 節で明文化、コードレビュー基準。
  • コード層 — 除外リスト (noukaweb / hojyokin-portal / biz.stayway 等) を scripts/ingest/* でフィルタ。
  • 運用層 — nightly source liveness scan で URL の生存確認、集約サイトの domain が混入していないか検査。

結果: 全 13,578 programs 行のうち 99%+ が一次資料 URL + fetched_at を保持し、残る 12 件 (CMS 不在の小規模自治体) も明示的に「一次 URL 不足」とフラグしています。15 系統のデータソース一覧 を参照。

差別化軸: 帝国DB / 東商リサーチとの違い

出典 URL は全て公開、 一次資料に直接 link する。

大手信用調査会社は内部評点 (信用評点・信頼度ランク) を商品の中心に据え、出典は社内 confidential。当社は逆方向で、評点ではなく 原典 URL そのものを商品の中心に据えています。

帝国DB / 東商リサーチ / 集約サイト / 税務会計AI の出典開示ポリシー比較
項目 帝国DB / 東商リサーチ 集約サイト (noukaweb 等) 税務会計AI
商品の中心 独自評点 (信用評点・信頼度ランク) 収集ボリュームと SEO 原典 URL + 構造化メタデータ
出典 URL の公開 非公開 (社内 confidential) 原典 URL なし or 自社中継 URL 全件公開 (am_source.source_url)
取得日時の公開 非公開 非公開 or 不正確 全件公開 (fetched_at + last_verified)
ライセンスの公開 独自データのため適用されず 明示なし 全件公開 (am_source.license、6 値)
cite chain の機械可読 非対応 非対応 GET /v1/am/programs/{id}/sources JSON
集約サイト採用 (サービス領域が異なる) 採用しているケースあり 採用しない (除外リスト + コード強制)
価格モデル 会員制 + 個別見積 無料 (アフィリエイト) ¥3/リクエスト 完全従量、匿名 50/月 free

この方向性は当社の使命に直結しています: 「ChatGPT / Claude が日本の制度を語るとき、出典を捏造せずに語れる」状態にすること。AI が API を叩いた時点で原典 URL に直接 link できる、というのは「評点を AI に見せる」とは目指す姿が違います。

適用される全データセット

cite chain は 9 データセット全てに同じ枠組みで適用されます:

  • programs (10,790 検索可) — 公募要領 PDF + 事務局サイト
  • case_studies (2,286) — 各省庁 / JST 公開採択結果
  • loan_programs (108) — 公庫 / 民間金融の公開ページ
  • enforcement_cases (1,185) — 各省庁公表の処分公報
  • laws (本文 154 + メタ 9,484) — e-Gov 法令データ提供システム (CC BY 4.0)
  • court_decisions (2,065) — 知財高裁 / 公開判決
  • bids (362) — NEXCO / JR / UR / 各発注機関
  • invoice_registrants (13,801 delta) — 国税庁適格請求書発行事業者公表 (PDL v1.0)
  • tax_rulesets (35) — 国税庁通達

各データセットの行数と最終取得日は データ鮮度ダッシュボード で live 確認できます。

削除請求 (Right to Be Forgotten)

適格請求書発行事業者公表データ等で「公表終了済みなのに掲載されている」「個人事業主として 屋号公表を取り下げた」等の削除請求は、info@bookyou.net 宛に法人番号 / 登録番号 / 屋号などを添えて連絡してください。原則 48 時間以内 (営業日) に削除または非表示処理を行います。詳細は プライバシーポリシー。
