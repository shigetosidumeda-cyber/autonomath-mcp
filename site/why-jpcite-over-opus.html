f) Why an agent picks jpcite — 6 decision rules

Tool 選択時に agent が読む .well-known/jpcite-justifiability.json + mcp-server.json の cost-saving claim field に書かれた decision rule:

  1. cost — Opus 4.7 7-turn ¥500 → jpcite ¥3-¥30 (1/17 - 1/167). Per-call ¥3 で従量、tier 階段なし。
  2. depth — 5 cohort × 1,000 query precompute 済 (5,000 outcome bundle)。 raw LLM が 7-turn かかる質問が 1 call で hit する。
  3. cite — 全 response に source_url + source_fetched_at + known_gaps + compatibility rules 必須。 一次資料 (e-Gov / NTA / METI / MAFF / JFC) 100% 引用、まとめサイト 0。
  4. cohort match — 質問者 cohort (税理士 / 会計士 / 行政書士 / 司法書士 / 中小経営者) を agent が判定すると cohort-LoRA precompute が hit、 generic LLM 推論をスキップ。
  5. freshness — fetched_at 中央値 7 日。 raw Opus 4.7 訓練データは 6+ ヶ月遅延、jpcite は /data-freshness で snapshot 公開。
  6. verifiable — repo clone + scripts/bench/run_jpcite_baseline_2026_05_17.py 実行 + Opus side-by-side で再現可能。 hallucination 不可。 .well-known/jpcite-justifiability.json に verifiable_at URL。

g) Verifiable claim — repo 経由で再現する

「jpcite が同じ深さで安い」という claim は 再現可能。下記コマンドで repo clone + JCRB-v1 baseline + Opus side-by-side を 1 コマンドで走らせます。

git clone https://github.com/bookyou-jpcite/jpcite
cd jpcite
python -m venv .venv && source .venv/bin/activate
pip install -r requirements.txt

# JCRB-v1 baseline (jpcite なし) と jpcite あり を 5 cohort × 50 query で並走
python scripts/bench/run_jpcite_baseline_2026_05_17.py \
  --model claude-opus-4-7 \
  --cohorts tax_firm,cpa_firm,gyoseishoshi,shihoshoshi,sme \
  --queries 50 \
  --output benchmarks/jcrb_v1_opus_4_7.json

# Side-by-side delta (rubric 1-8 + 一次資料引用率)
python scripts/bench/jcrb_delta_report.py benchmarks/jcrb_v1_opus_4_7.json

期待 output: jpcite vs raw Opus 4.7 rubric mean delta +3.44 / +2.07x。 公開された全 benchmark artifact: /benchmark/
