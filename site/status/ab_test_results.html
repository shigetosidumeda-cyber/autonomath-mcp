Public process

新しい A/B テストを追加する公開運用手順

  1. 公開 assignment endpoint に対象 test_id を登録します。
  2. 公開ページ側で割当結果を読み、返却 group で variant を出し分けます。
  3. completion event を集計し、公開結果 JSON に反映します。
  4. 集計済みの公開結果 JSON を毎日更新し、本ページに反映します。
  5. 95% 信頼で winner が出たら旧 variant をリポジトリから削除し、新 variant 一本化。
