運用方針: jpcite は auto-translate を一切しません。 コーパスに乗る訳文は全て 一次資料 (e-Gov 日本法令外国語訳 CC-BY 4.0 / 各官庁 EN sub-path / JETRO zh-cn/zh-tw/kr 等) からの fetch のみです。 取れなかった law / program は data/*_review_queue.csv に積まれ、operator が手動で 1 次資料 URL を貼り付けた行のみ next cron で promote されます。 LLM API は呼びません (memory: feedback_no_operator_llm_api)。

Language coverage

Manual review queue 入口

review queue 経路: 各 fill cron は primary EN/ZH/KO mirror を一通り叩いた後、 取れなかった entry を CSV に追記します。 operator は CSV の operator_url 列に 1 次資料 URL を埋め、 次回 cron が CSV を読んで該当 row を fetch → DB へ書き込みます。 何も書かない限り 絶対に DB に load されない ので、 誤訳・自動翻訳のリスクはゼロです。

REST API

プログラム的に取得する場合は GET /v1/translation/status?lang=en / ?lang=zh / ?lang=ko / ?lang=all を叩いてください。 1 req = 1 unit (¥3) でメータリングされます。 

$ curl https://api.jpcite.com/v1/translation/status?lang=en
{
  "languages": {
    "en": {
      "laws": {"total": 6493, "filled": ..., "coverage_pct": ...},
      "articles": {"total": 353278, "filled": ..., "coverage_pct": ...},
      "programs": {"total": 11601, "filled": ..., "coverage_pct": ...},
      "review_queue_pending": ...,
      "last_refreshed_at": "..."
    }
  },
  "generated_at": "...",
  "disclaimer": "..."
}
データソース: autonomath.db am_law + am_law_article + am_law_translation_review_queue, jpintel.db programs. クロール頻度: weekly (.github/workflows/multilingual-weekly.yml) + monthly audit (multilingual-monthly-audit.yml). 著作権: 各 1 次資料の license を尊重 (e-Gov JLT は CC-BY 4.0, 官庁 EN サイトは政府著作物原則, JETRO public-domain). 本ダッシュボードは scripts/cron/generate_translation_dashboard.py で再生成されます。
