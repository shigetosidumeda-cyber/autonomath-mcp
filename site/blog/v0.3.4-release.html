jpcite v0.3.4 — 5 つの新機能

エージェント運用・経理稟議・第三者監査・国際展開・OSS 拡張の 5 領域で機能追加。 主な評価軸は verify 時間の推定短縮 と 監査可能引用の正確性 です。測定条件に基づく参考値は公開ベンチに分離しており、個別の削減時間・費用対効果を保証しません。 token 削減は副次効果で、 raw token math だけでは ¥3/billable unit を償却できないケースがあります。 料金は ¥3/billable unit のまま据え置き、tier なし。

スタートガイド → 変更履歴一覧 →

新機能 (5)

  1. 1. Bulk endpoint

    1 リクエストで 100 件まで一括取得 — 100 件の手動 verify が 1 件の audit-trail に

    エージェントが複数の制度を並列照合する用途に最適化。1 回の API 呼び出しで最大 100 件の Evidence Packet を一括取得し、 出典 URL + 取得時刻 + content_hash を 1 つの監査ログに揃えられます (verify 時間が線形に削減)。新規エンドポイント: POST /v1/evidence/packets/batch

  2. 2. Stripe クレジット前払

    ¥30万・¥100万・¥300万を一括前払

    経理稟議が通りやすい一括前払型のクレジット購入を追加。年度予算に組み込みやすく、月次請求書の発生を抑制できます。 verify 時間 (月 ~50h ≈ ¥400,000 相当の節約) が稟議の主論点であり、 jpcite 月額 ~¥40,000 はその対価。新規エンドポイント: POST /v1/billing/credit/purchase

  3. 3. Merkle 監査チェーン

    取得済み行が改ざんされていないことを第三者検証 — citation 正確性の決定打

    税理士・監査法人向けに、Evidence Packet 単位で Merkle 証明を発行。配信後にデータが書き換わっていないことを暗号学的に検証できるため、 監査時の人手 cross-check (~30 分/件) を不要にします。 citation 信頼性が jpcite の真の価値軸 (token 削減ではない)。新規エンドポイント: GET /v1/audit/proof/{evidence_packet_id}

  4. 4. English wedge

    在日米英資 SaaS / FDI 向け英訳サーフェス

    e-Gov 英訳法令と租税条約を含む 5 エンドポイントを /v1/en/* 配下に新設。外資系顧客のオンボーディング、外国親会社向けレポートなどで利用できます。

  5. 5. Reference agent npm

    @jpcite/agents — fork-and-customize 可能な 5 種の参考エージェント

    jpcite を呼び出すエージェントの最小構成サンプルを npm パッケージとして公開。fork して自社業務に合わせてカスタマイズすることを前提にした 5 種類のスターターエージェントを同梱しています。

その他

  • 1,034 都道府県 × 業種ランディングページ — 47 都道府県 × 22 業種でカバレッジを拡張。検索流入経路を強化。
  • ブラウザ拡張 — 表示中ページの法人番号・制度名から jpcite を直接呼び出すブラウザ拡張を公開。
  • VS Code 拡張 — エディタ内で制度・法令を直接引けるサイドバー型 VS Code 拡張を公開。
  • JCRB-v1 公開ベンチマーク — Japanese Citation Retrieval Benchmark v1 を公開。日本語の制度・法令引用の精度比較に利用できます。

料金は不変

¥3/billable unit 純従量 (税込 ¥3.30)、tier なし、匿名 3 req/日 無料 (IP 単位、JST 翌日 00:00 リセット)。 v0.3.4 でもこの料金体系は変更ありません。前払クレジット購入は同じ billable unit レートを前払で確保するもので、単価は同一です。
